## 章节大纲

### 开头引入

- 切入点：你跟 ChatGPT 聊了一下午，第二天打开新对话，它不认识你了。你的智能体也一样——对话结束，一切归零
- 提出问题：怎么让 AI 记住"你是谁、上次聊了什么、你喜欢什么"？
- 本文思路：从最简单的对话历史管理，一步步搭到带长期记忆的"懂你"助手
- 和系列的衔接：智能体那篇提到的 messages 列表就是最原始的记忆，这篇来升级它

### 一、智能体的记忆到底是什么？

- 类比人的记忆系统：工作记忆（当前在想的事）、短期记忆（今天发生的事）、长期记忆（你是谁、经历过什么）
- 智能体的三层记忆对应关系：
  - **工作记忆**：当前这一轮对话的 messages 列表
  - **短期记忆**：一次会话内的对话历史
  - **长期记忆**：跨会话的用户偏好、历史摘要、知识积累
- 为什么 messages 列表不够用：token 限制、成本、上下文窗口溢出

### 二、最简方案：滑动窗口（保留最近 N 轮）

**要点：**
- 原理：只保留最近 N 轮对话，超出的直接丢掉
- 优点：实现简单，token 可控
- 缺点：对话长了之后前面说过什么全忘了

**核心代码片段：**

```python
def sliding_window_memory(messages, max_turns=10):
    """滑动窗口：只保留最近 N 轮对话"""
    system_msg = [m for m in messages if m["role"] == "system"]
    conversation = [m for m in messages if m["role"] != "system"]
    # 每轮 = 一组 user + assistant，保留最近 max_turns 轮
    if len(conversation) > max_turns * 2:
        conversation = conversation[-(max_turns * 2):]
    return system_msg + conversation
```

**模拟运行效果：**
```
对话第 1 轮：用户说喜欢吃川菜 → AI 记住了
对话第 5 轮：用户问推荐餐厅 → AI 推荐川菜馆 ✅
对话第 15 轮（窗口=10）：用户问推荐餐厅 → AI 不知道你喜欢川菜了 ❌
```

### 三、进阶方案：摘要压缩（用 LLM 给历史做总结）

**要点：**
- 原理：对话超过阈值后，用大模型把早期对话压缩成一段摘要，只保留关键信息
- 摘要 + 最近 N 轮 = 完整上下文
- 压缩比：20 轮对话 → 一段 200 字摘要，token 节省 80%+

**核心代码片段：**

```python
SUMMARIZE_PROMPT = """请将以下对话历史压缩成一段简洁的摘要，保留：
1. 用户提到的关键信息（姓名、偏好、需求）
2. 已经做出的重要决定
3. 待办事项或未解决的问题
丢弃闲聊和重复内容。

对话历史：
{conversation}

请输出摘要（控制在 200 字以内）："""

class SummaryMemory:
    def __init__(self, max_recent=6, summarize_threshold=12):
        self.messages = []
        self.summary = ""
        self.max_recent = max_recent
        self.summarize_threshold = summarize_threshold

    def add(self, role, content):
        self.messages.append({"role": role, "content": content})
        if len(self.messages) > self.summarize_threshold:
            self._compress()

    def _compress(self):
        old_msgs = self.messages[:-self.max_recent]
        old_text = "\n".join([f"{m['role']}: {m['content']}" for m in old_msgs])
        # 调用 LLM 做摘要
        self.summary = ask_llm(SUMMARIZE_PROMPT.format(conversation=old_text))
        self.messages = self.messages[-self.max_recent:]

    def get_messages(self, system_prompt):
        context = f"{system_prompt}\n\n【之前的对话摘要】{self.summary}" if self.summary else system_prompt
        return [{"role": "system", "content": context}] + self.messages
```

**模拟运行效果：**
```
对话 1-10 轮：正常聊天，用户提到名字叫小明，喜欢川菜，下周要出差上海
第 12 轮触发压缩 → 摘要："用户名小明，偏好川菜，计划下周出差上海"
第 20 轮用户问"我下周去哪来着？" → AI 从摘要中找到"出差上海" ✅
```

### 四、长期记忆：跨会话的向量化存储

**要点：**
- 问题：上面的方案只在单次会话内有效，关掉窗口就没了
- 解决：把关键信息持久化到向量数据库（ChromaDB），下次打开能找回来
- 存什么：用户偏好、重要决定、历史问答摘要
- 怎么取：每次对话开始，用用户的问题去向量库搜相关的历史记忆

**核心代码片段：**

```python
import chromadb

class LongTermMemory:
    def __init__(self, user_id):
        self.client = chromadb.PersistentClient(path="./memory_db")
        self.collection = self.client.get_or_create_collection(
            name=f"user_{user_id}"
        )

    def save_memory(self, content, memory_type="preference"):
        """保存一条记忆"""
        self.collection.add(
            ids=[f"mem_{int(time.time())}"],
            documents=[content],
            metadatas=[{"type": memory_type, "timestamp": str(time.time())}],
            embeddings=[get_embedding(content)]
        )

    def recall(self, query, top_k=3):
        """根据当前对话回忆相关记忆"""
        results = self.collection.query(
            query_embeddings=[get_embedding(query)],
            n_results=top_k
        )
        return results["documents"][0] if results["documents"] else []

    def build_context(self, query, system_prompt):
        """把回忆到的记忆注入 system prompt"""
        memories = self.recall(query)
        if memories:
            memory_text = "\n".join([f"- {m}" for m in memories])
            return f"{system_prompt}\n\n【关于这位用户的记忆】\n{memory_text}"
        return system_prompt
```

**模拟运行效果：**
```
第一天对话：用户说"我叫小明，是前端开发，最近在学 Rust"
→ 保存记忆：["用户名小明", "职业前端开发", "正在学习Rust"]

三天后新对话：用户问"有什么适合我的技术文章推荐吗？"
→ 回忆：检索到"前端开发"、"正在学习Rust"
→ AI 推荐 Rust 入门 + 前端转全栈的文章 ✅（跨会话记住了）
```

### 五、工作记忆：Scratchpad 模式

**要点：**
- 问题：有些信息不是用户说的，是智能体自己推理出来的中间结论
- 解决：给智能体一块"草稿纸"（Scratchpad），让它自己记笔记
- 用法：在 system prompt 里加一块可读写的区域，每轮更新

**核心代码片段：**

```python
SCRATCHPAD_PROMPT = """你有一块草稿纸，用来记录重要的中间信息。
每次回答后，更新你的草稿纸内容。

【当前草稿纸】
{scratchpad}

格式：在回答末尾用 [SCRATCHPAD] 标记，写下需要记住的内容。
"""

class WorkingMemory:
    def __init__(self):
        self.scratchpad = "（空）"

    def update_from_response(self, response_text):
        if "[SCRATCHPAD]" in response_text:
            parts = response_text.split("[SCRATCHPAD]")
            self.scratchpad = parts[1].strip()
            return parts[0].strip()  # 返回给用户的部分
        return response_text
```

### 六、整合：一个完整的记忆系统架构

**要点：**
- 把四种记忆组合在一起的完整架构图
- 数据流：用户消息进来 → 工作记忆处理 → 短期摘要 → 长期存储 → 下次回忆
- 代码片段：一个 `MemoryManager` 整合四层记忆

**核心代码片段：**

```python
class MemoryManager:
    def __init__(self, user_id):
        self.summary_mem = SummaryMemory()
        self.long_term = LongTermMemory(user_id)
        self.working = WorkingMemory()

    def build_prompt(self, user_query, base_system_prompt):
        # 1. 从长期记忆回忆相关信息
        prompt = self.long_term.build_context(user_query, base_system_prompt)
        # 2. 加上工作记忆（草稿纸）
        prompt = SCRATCHPAD_PROMPT.format(scratchpad=self.working.scratchpad)  + "\n" + prompt
        # 3. 返回带摘要的 messages
        return self.summary_mem.get_messages(prompt)

    def after_response(self, user_msg, ai_response):
        # 更新短期记忆
        self.summary_mem.add("user", user_msg)
        self.summary_mem.add("assistant", ai_response)
        # 更新工作记忆
        clean_response = self.working.update_from_response(ai_response)
        # 提取关键信息存入长期记忆（可用 LLM 判断是否值得记住）
        self._maybe_save_long_term(user_msg)
        return clean_response
```

### 七、对比总结

```
| 记忆类型     | 原理             | 持久性   | 适合场景           | 成本     |
|-------------|-----------------|---------|-------------------|---------|
| 滑动窗口     | 保留最近 N 轮     | 会话内   | 简单对话           | 最低     |
| 摘要压缩     | LLM 总结历史     | 会话内   | 长对话、多步任务     | 中等     |
| 长期记忆     | 向量库持久化      | 跨会话   | 个性化助手         | 稍高     |
| 工作记忆     | Scratchpad 草稿  | 单次任务  | 复杂推理、多步计划   | 最低     |
```
