# GPT 是怎么一个字一个字"蹦"出来的？

你盯着屏幕，看 ChatGPT 的光标一闪一闪。它像一个人在打字，又像在思考。每一个字蹦出来，都带着某种节奏感。

但真相是——它不是在"打字"，也不是在"思考"。

它只是在玩一个超级复杂的接龙游戏。

---

## 一、大模型的核心任务：预测下一个字

GPT 干的事，本质上就一件：**猜下一个字是什么**。

你输入"今天天气"，它猜"很"；你输入"今天天气很"，它猜"好"；你输入"今天天气很好"，它猜"，"或者"。"……就这么一路猜下去，直到它觉得该停了。

这有个专业术语叫 **next token prediction**（下一个词元预测）。听起来高大上，其实就是接龙。

小时候玩过成语接龙吧？别人说"一心一意"，你接"意气风发"。大模型干的也是这个，只不过它的"词典"是几十万个 token，它的"接龙规则"是从海量文本里学出来的概率分布。

所以下次看到 ChatGPT 慢悠悠地往外蹦字，别以为它在绞尽脑汁——它只是在算：根据前面这一大串，下一个最可能是什么？

---

## 二、Token：模型眼中的"最小单位"

说到 token，很多人会想：不就是"字"吗？

不完全是。**Token** 是模型真正处理的最小单位。中文通常一个字一个 token，但英文一个词可能被拆成好几个。比如 "unbelievable" 可能变成 "un"、"believ"、"able" 三块。

对理解"生成过程"来说，你只需要记住：**模型每次只生成一个 token，然后基于"已有内容 + 这个 token"再生成下一个**。它看不见整段话，只能看见已经写出来的部分。Token 的细节（比如 BPE 分词、中英文差异）可以留到专门讲 token 的文章里，这里知道"最小单位"就够了。

---

## 三、生成过程：一个反直觉的真相

模型怎么把一整段话"写"出来？

答案是：**一个接一个，像推多米诺骨牌**。

你给它一个开头，它算出第一个 token 的概率分布，从中选一个，append 到序列里；然后拿着"开头 + 第一个 token"再去算第二个，再选一个，再 append……如此往复，直到生成"结束"标记，或者达到长度上限。

这种"每次只生成一个，生成完再基于已有内容生成下一个"的方式，叫 **自回归生成**。就像推多米诺骨牌，推倒第一张，第二张跟着倒，第三张再跟着……一张接一张，没有回头路。

**这里有个反直觉的发现：它不会一次性想好整段话。**

它写到一半，也不知道自己接下来要说什么。它没有"大纲"，没有"规划"，只有"根据前面这些，下一个最可能是什么"的即时计算。就像你写作文时，写第一句的时候脑子里有开头，写第二句的时候会参考第一句——但大模型更极端：它连"脑子里"都没有，只有"前面已经写出来的"。

换句话说：它写"因为"的时候，还没决定后面要接"所以"还是"但是"。它写"今天"的时候，也不知道这句会变成"今天天气很好"还是"今天不想上班"。每一个字都是临场发挥，没有剧本。

**金句**：人类写作是"想好了再写"，大模型是"写出来了才知道自己写了啥"。它没有预演，没有草稿，只有实时的概率计算。

所以它不可能"想好再输出"。它没有"想好"这个动作，只有"算下一个"这个动作。你看到的流式效果，就是它真实的生成节奏。

## 四、概率和采样：为什么同一个问题每次回答不一样？

你问 ChatGPT"推荐一本好书"，第一次它说《三体》，第二次可能说《百年孤独》，第三次又换了。

为啥？因为模型输出的不是"一个答案"，而是**每个可能 token 的概率分布**。

比如下一个位置，模型算出来可能是："《" 30%，"我" 25%，"推" 20%……它不会直接选概率最高的那个，而是按概率**抽样**。抽到谁就输出谁。所以同样的输入，每次抽样结果可能不同，回答就变了。

这就像抽奖。箱子里有 100 个球，"《" 30 个，"我" 25 个，"推" 20 个……每次伸手摸一个，摸到啥就输出啥。概率高的球多，被摸到的机会大，但不是绝对的。所以"推荐《三体》"和"推荐《百年孤独》"可能都是合理接龙，只是每次抽到的球不一样。

---

## 五、Temperature：AI 的"性格开关"

那能不能控制这种"随机性"？

能。有个参数叫 **temperature**，可以理解成 AI 的"胆子大小"。

**Temperature 低**（比如 0.2）：概率分布会被"压扁"，高概率的更高，低概率的更低。模型会倾向于选那些"最稳妥"的词，输出更保守、更可预测、更像"标准答案"。就像一个人被要求"只说最安全的话"。

**Temperature 高**（比如 1.0 或更高）：概率分布更"平"，各种词都有机会被抽到。模型会更放飞自我，创意更多，但也更容易跑偏、胡说八道。就像一个人喝了点酒，什么都敢说。

**实际调参的体验**：写代码、做数学题、提取数据，把 temperature 调到 0.2 甚至 0.1，输出会稳定得像"标准答案"；写诗、头脑风暴、创意文案，调到 0.8 或 1.0，你会发现它开始冒出你没想到的句子。这不是玄学，就是控制"抽样时有多敢选那些概率没那么高的选项"。

有个比喻很贴切：temperature 就像"创意阀门"。拧紧了，出来的都是最稳妥的词；拧松了，各种稀奇古怪的选项都有机会冒头。很多人在调 API 时忽略这个参数，结果发现"有时候很准，有时候乱说"——多半是 temperature 默认值偏高，试试调到 0.3，世界会安静很多。

## 六、Top-K 和 Top-P：控制 AI 的"词汇范围"

除了 temperature，还有两个常用手段：**Top-K** 和 **Top-P**。

它们解决的问题是：概率分布里可能有几万个候选 token，总不能真从几万里随机抽吧？那样的话，那些概率极低的奇怪词也会有机会蹦出来——比如突然冒出一个"魑魅魍魉"。

**Top-K** 的意思是：只考虑概率最高的 K 个 token，其他的直接扔掉。比如 Top-K=50，那就在前 50 名里抽样。这样就把 AI 的"词汇范围"限制在一个相对靠谱的圈子里。

**Top-P**（也叫 nucleus sampling）：从概率最高的 token 开始累加，直到累计概率达到 P，然后只在这个集合里抽样。比如 Top-P=0.9，就是"取到累计 90% 概率的那批 token"。

可以类比成选餐厅。Top-K 是"只考虑点评榜前 50 名"；Top-P 是"只考虑点评榜上覆盖 90% 好评的那一批"。都能把选择范围收窄，让输出更稳定。

为什么需要两个？因为 Top-K 是"固定名额"，Top-P 是"动态名额"。有时候概率分布很集中，前 10 个 token 就占了 95%，那 Top-P=0.9 可能只考虑 5 个；有时候分布很散，前 50 个才凑够 90%，那 Top-P=0.9 就考虑 50 个。Top-P 更灵活，所以很多 API 默认只用 Top-P，不用 Top-K。

**调参小技巧**：做严肃任务（代码、数据、翻译），Top-P 调到 0.3-0.5，Top-K 调到 20-40，输出会非常"乖"；做创意任务，可以放宽到 Top-P=0.9、Top-K=100，让模型有更多发挥空间。很多 API 默认 Top-P=1.0，意味着"不限制"——如果你发现输出经常跑偏，试试把它调小。

这三个参数（temperature、Top-K、Top-P）常常一起调。可以这么理解：temperature 控制"敢不敢选冷门"，Top-K/Top-P 控制"冷门选项的池子有多大"。池子小 + 胆子小 = 输出非常稳定；池子大 + 胆子大 = 创意爆棚，但也可能翻车。

## 七、Beam Search：顺便提一嘴

还有一种选 token 的方式叫 **Beam Search**（束搜索）：不是每次只保留一条路径，而是同时保留 top-K 条，每步都扩展，保留最好的。就像走迷宫时"同时看好几条路，随时准备换道"。

不过 GPT 这类模型在开放对话里通常不用 Beam Search，而是用前面说的采样。Beam Search 更常见于翻译、摘要这类"有标准答案"的任务。普通用户不用太关心这个——知道有这么回事就行。

---

## 八、流式输出的秘密

你看到 ChatGPT 一个字一个字蹦，背后其实是**流式输出**（streaming）。

模型生成 token 的时候，本来就是一个一个算的。算出一个，就可以立刻推给前端展示，不必等整段话都算完。这就是流式——边算边传，边传边显示。技术实现上，就是服务器每算完一个 token 就通过 HTTP 或 WebSocket 推给客户端，客户端收到就渲染，形成"打字"的视觉效果。

**一个有趣的细节**：你看到的"打字效果"不是刻意做的动画，而是**真实反映了模型的生成节奏**。每个字蹦出来的时候，模型确实刚算完那个 token。你看到的，就是它"思考"的实况转播。

如果不用流式呢？服务器会等模型把整段话都生成完，一次性返回。那时候你看到的就是：转圈圈，转圈圈，然后哗一下整段话全出来。体验差很多——你甚至会怀疑它是不是在偷懒，其实它一直在算，只是没让你看见。早期的很多 AI 产品就是这样，所以用户总觉得"卡"。流式输出普及之后，那种"在等我"的感觉才真正建立起来。

所以下次看到光标闪烁，你可以想象：网络的另一端，有一台机器正在疯狂地算概率、抽样、append，一个 token 接一个 token，把"下一个最可能"推到你的屏幕上。

**还有一个细节**：为什么有时候字蹦得快，有时候慢？因为不同的 token 计算量不同——生成长词、生僻字、或者模型"犹豫"（多个选项概率接近）时，会稍微慢一点。你看到的节奏，其实是模型"决策难度"的实时反馈。下次注意观察：当它卡顿一下再蹦出下一个字，多半是那个位置有好几个势均力敌的选项，它在"纠结"。

## 九、知道这些有什么用？

**第一，调参数的时候心里有数。** 做代码生成、数据提取，把 temperature 调低、Top-P 调小，输出更稳定；做创意写作、头脑风暴，把 temperature 调高，让模型更敢想。你不是在瞎试，而是在控制"概率抽样"的松紧。下次遇到"输出不稳定"的问题，先想想：是不是该拧紧创意阀门了？

**第二，理解 AI 的"怪癖"。** 为什么同一个问题问两遍答案不一样？因为采样。为什么有时候会胡说八道？因为 temperature 太高，低概率的离谱选项也被抽到了。为什么长文本容易跑题？因为自回归是"走一步看一步"，没有全局规划，走着走着就偏了。知道这些，你就不会怪它"怎么又变了"，而是会想"我是不是该把 temperature 调低一点"。

**第三，设计更好的 prompt。** 你知道模型是"根据上文猜下一个"，所以你的 prompt 就是在给它"上文"。上文越清晰、越具体，它猜得越准。含糊的指令等于含糊的上文，输出自然也就飘忽不定。给它一个清晰的开头，它才能接出你想要的龙。

**第四，少一点"它应该知道"的期待。** 它没有记忆，没有常识库，只有从训练数据里学到的统计规律。它"知道"的东西，都是"下一个 token 最可能是什么"的概率体现。理解了这一点，你就不会在它犯低级错误时暴跳如雷——它只是在接龙，接错了而已。

---

## 十、尾声

下次再看到 ChatGPT 的光标一闪一闪，你可以会心一笑。

它不是在装深沉，也不是在模仿人类打字。它只是在玩一场规模巨大的接龙游戏——没有预谋，没有全局视野，只有"根据前面这些，下一个最可能是什么"的即时计算。

你看到的每一个字，都是它刚算出来的"下一个"。没有剧本，没有彩排，只有实时的概率和抽样。

知道规则，才能玩得更好。而玩得更好的意思，不是去拆穿它的把戏，而是学会跟它配合——给它清晰的上文，调对合适的参数，让它接出你想要的龙。

而最有趣的是：即使知道了规则，看着它一个字一个字蹦出来，依然会有一点奇妙的错觉——仿佛对面真的有人在想。那种"等它说完"的期待感，那种光标闪烁带来的节奏感，会让人不自觉地把它当成对话者。

也许，这就是为什么我们愿意跟一个"只会接龙"的机器聊天。它没有心，但它的输出方式，恰好模拟了"思考"的样子。而我们，恰好需要这种错觉。

光标闪烁，字一个一个蹦。你知道它在算什么，但你还是会等它说完。

这大概就是人机对话最微妙的地方：理性上你拆穿了把戏，感性上你还是把它当成了对话者。

下次聊天时，不妨带着这份"知道"——你会看得更清楚，也会玩得更从容。

而那个一闪一闪的光标，会多一层意味：你知道它在算什么，但你还是会等。这就是接龙游戏的魅力。
