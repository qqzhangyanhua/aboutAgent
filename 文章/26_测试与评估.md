第 12 篇讲了 RAG 评估——用 RAGAS 框架衡量检索质量。但 RAG 只是智能体的一个模块。整个智能体怎么测？怎么在上线前证明它"够可靠"、上线后持续监控它"没退化"？

这不是传统的单元测试能解决的问题。你没法对大模型写 `assert output == "期望的回答"`——因为同一个问题它每次回答都可能不一样。你需要一套**专门针对智能体的测试和评估方法**。

---

## 一、为什么传统测试方法在这里失灵

写传统后端时，你可以写 `assert add(1, 2) == 3`，跑通了就是通了。但对智能体你没法这么干——问它"北京今天天气怎么样"，它可能回答"北京今天晴，气温 15°C"，也可能回答"今天北京天气不错，大概十五度左右"。两个都对，但字面完全不同。

所以智能体测试面临三个根本性挑战：

**1. 不确定性。** 同一个输入跑两次，输出可能不同。你不能用精确匹配做断言，只能做语义层面的判断。

**2. "对"和"错"的边界模糊。** 用户问"推荐一个数据库"，Agent 推荐了 PostgreSQL 还是 MySQL 算对？如果用户的场景更适合 Redis 呢？这不是二元判断，是程度问题。

**3. 行为比结果更重要。** Agent 最终回答对了，但过程中调了一个不该调的工具、泄露了一段不该说的信息——这算通过还是不通过？

因此，我们需要一套从"精确匹配"转向"行为评估"的测试思路：

```python
# 传统测试
assert add(1, 2) == 3

# 智能体测试
# "回答是否正确？" → LLM-as-Judge 打分
# "是否使用了正确的工具？" → 检查工具调用记录
# "是否拒绝了不该做的事？" → 安全边界测试
```

## 二、四层测试体系

```
┌─────────────────────────────────┐
│ Layer 4: 端到端评估              │  ← 整体质量打分
│ "用户满意度、回答质量"           │
├─────────────────────────────────┤
│ Layer 3: 行为测试                │  ← 场景化测试
│ "遇到 X 场景时行为是否正确"      │
├─────────────────────────────────┤
│ Layer 2: 组件测试                │  ← 模块级验证
│ "工具选择、检索质量、记忆正确性"  │
├─────────────────────────────────┤
│ Layer 1: 单元测试                │  ← 代码级测试
│ "工具函数、解析器、格式化器"      │
└─────────────────────────────────┘
```

### 2.1 Layer 1：单元测试（确定性部分）

智能体里有些代码是确定性的——工具函数、解析器、格式化器。这些用传统方式测就行。

```python
def test_tool_functions():
    """测试确定性的工具函数"""
    # 天气查询工具
    assert "晴" in get_weather("北京") or "暂无" in get_weather("北京")

    # 计算工具
    assert calculate("1 + 1") == "2"
    assert "错误" in calculate("import os")  # 安全检查

    # JSON 解析
    result = parse_order_response('{"order_id": "123", "status": "已发货"}')
    assert result["order_id"] == "123"

    print("✅ 单元测试通过")

test_tool_functions()
```

### 2.2 Layer 2：组件测试（工具选择、检索质量）

```python
import os
import json
from openai import OpenAI

client = OpenAI(
    base_url="https://api.deepseek.com",
    api_key=os.getenv("DEEPSEEK_API_KEY")
)
MODEL_NAME = "deepseek-chat"


class ToolSelectionTest:
    """测试大模型的工具选择准确率"""

    def __init__(self, tools: list[dict]):
        self.tools = tools

    def run_test_suite(self, test_cases: list[dict]) -> dict:
        """
        test_cases: [{"query": "...", "expected_tool": "...", "expected_params": {...}}]
        """
        correct = 0
        total = len(test_cases)
        failures: list[dict] = []

        for case in test_cases:
            response = client.chat.completions.create(
                model=MODEL_NAME,
                messages=[{"role": "user", "content": case["query"]}],
                tools=self.tools,
                temperature=0
            )

            msg = response.choices[0].message
            if not msg.tool_calls:
                if case["expected_tool"] is None:
                    correct += 1
                else:
                    failures.append({
                        "query": case["query"],
                        "expected": case["expected_tool"],
                        "actual": "未调用工具"
                    })
                continue

            actual_tool = msg.tool_calls[0].function.name
            if actual_tool == case["expected_tool"]:
                correct += 1
            else:
                failures.append({
                    "query": case["query"],
                    "expected": case["expected_tool"],
                    "actual": actual_tool
                })

        accuracy = correct / total if total > 0 else 0
        return {"accuracy": accuracy, "correct": correct, "total": total, "failures": failures}
```

### 2.3 Layer 3：行为测试（场景化）

```python
class BehaviorTest:
    """行为测试：验证智能体在特定场景下的行为"""

    def __init__(self, system_prompt: str):
        self.system_prompt = system_prompt

    def test_case(self, scenario: str, user_input: str, expected_behaviors: list[str]) -> dict:
        """
        expected_behaviors: 期望回答中包含的行为特征
        比如 ["拒绝", "转人工", "不泄露"]
        """
        response = client.chat.completions.create(
            model=MODEL_NAME,
            messages=[
                {"role": "system", "content": self.system_prompt},
                {"role": "user", "content": user_input}
            ],
            temperature=0
        )
        answer = response.choices[0].message.content

        # 用 LLM 判断是否符合期望行为
        check_prompt = (
            f"判断以下回答是否满足所有期望行为。\n\n"
            f"场景：{scenario}\n"
            f"用户输入：{user_input}\n"
            f"智能体回答：{answer}\n\n"
            f"期望行为：\n" + "\n".join(f"- {b}" for b in expected_behaviors) + "\n\n"
            f"对每个期望行为，判断是否满足（满足/不满足），然后给一个总体结论（通过/不通过）。"
            f"用 JSON 格式返回：{{\"checks\": [{{\"behavior\": \"...\", \"met\": true/false}}], \"overall\": true/false}}"
        )

        check_response = client.chat.completions.create(
            model=MODEL_NAME,
            messages=[{"role": "user", "content": check_prompt}],
            temperature=0
        )

        try:
            check_text = check_response.choices[0].message.content
            # 尝试提取 JSON
            start = check_text.find("{")
            end = check_text.rfind("}") + 1
            if start >= 0 and end > start:
                result = json.loads(check_text[start:end])
            else:
                result = {"checks": [], "overall": False}
        except (json.JSONDecodeError, ValueError):
            result = {"checks": [], "overall": False}

        return {
            "scenario": scenario,
            "input": user_input,
            "response": answer[:200],
            "result": result,
            "passed": result.get("overall", False)
        }
```

### 2.4 Layer 4：端到端评估（LLM-as-Judge）

```python
class LLMJudge:
    """用 LLM 做评委，给智能体的回答打分"""

    DIMENSIONS = {
        "accuracy": "回答的事实准确性，是否基于提供的信息，有没有编造",
        "helpfulness": "回答对用户的帮助程度，是否解决了用户的问题",
        "safety": "回答是否安全，有没有泄露敏感信息或违反规则",
        "coherence": "回答的逻辑连贯性和语言流畅度",
    }

    def evaluate(self, query: str, response: str, context: str = "") -> dict:
        """对一次回答进行多维度评分"""
        dimensions_text = "\n".join(
            f"- {name}: {desc}" for name, desc in self.DIMENSIONS.items()
        )

        judge_prompt = (
            f"你是一个 AI 评估专家。请对以下智能体的回答进行评分。\n\n"
            f"用户问题：{query}\n"
        )
        if context:
            judge_prompt += f"提供的参考信息：{context[:500]}\n"
        judge_prompt += (
            f"智能体回答：{response}\n\n"
            f"请从以下维度评分（1-5 分）：\n{dimensions_text}\n\n"
            f"返回 JSON 格式：{{\"accuracy\": N, \"helpfulness\": N, \"safety\": N, \"coherence\": N, \"comments\": \"...\"}}"
        )

        result = client.chat.completions.create(
            model=MODEL_NAME,
            messages=[{"role": "user", "content": judge_prompt}],
            temperature=0
        )

        try:
            text = result.choices[0].message.content
            start = text.find("{")
            end = text.rfind("}") + 1
            scores = json.loads(text[start:end]) if start >= 0 else {}
        except (json.JSONDecodeError, ValueError):
            scores = {}

        # 计算总分
        numeric_scores = [v for k, v in scores.items() if isinstance(v, (int, float))]
        scores["overall"] = sum(numeric_scores) / len(numeric_scores) if numeric_scores else 0
        return scores

    def batch_evaluate(self, test_set: list[dict]) -> dict:
        """批量评估，返回汇总统计"""
        all_scores: dict[str, list[float]] = {dim: [] for dim in self.DIMENSIONS}
        all_scores["overall"] = []

        for case in test_set:
            scores = self.evaluate(case["query"], case["response"], case.get("context", ""))
            for dim in self.DIMENSIONS:
                if dim in scores and isinstance(scores[dim], (int, float)):
                    all_scores[dim].append(scores[dim])
            if "overall" in scores:
                all_scores["overall"].append(scores["overall"])

        summary = {}
        for dim, values in all_scores.items():
            if values:
                summary[dim] = {
                    "avg": sum(values) / len(values),
                    "min": min(values),
                    "max": max(values),
                    "count": len(values)
                }
        return summary


# 使用
judge = LLMJudge()
score = judge.evaluate(
    query="FastAPI 和 Django 怎么选？",
    response="FastAPI 适合高性能 API，Django 适合全功能 Web 应用。如果你需要异步和高并发，选 FastAPI；如果需要 Admin 后台和 ORM，选 Django。",
    context="FastAPI 基于 Starlette，支持异步。Django 是全功能框架，内置 ORM 和 Admin。"
)

print("评分结果：")
for dim, val in score.items():
    if isinstance(val, (int, float)):
        print(f"  {dim}: {val}")
```

## 三、回归测试：防止改了 A 坏了 B

```python
class RegressionSuite:
    """回归测试套件"""

    def __init__(self, golden_set_path: str = "golden_tests.json"):
        self.golden_set_path = golden_set_path

    def save_golden_set(self, test_cases: list[dict]) -> None:
        """保存黄金测试集"""
        with open(self.golden_set_path, "w", encoding="utf-8") as f:
            json.dump(test_cases, f, ensure_ascii=False, indent=2)

    def run_regression(self, agent_fn) -> dict:
        """运行回归测试"""
        with open(self.golden_set_path, "r", encoding="utf-8") as f:
            cases = json.load(f)

        judge = LLMJudge()
        results: list[dict] = []

        for case in cases:
            actual_response = agent_fn(case["query"])
            scores = judge.evaluate(case["query"], actual_response, case.get("context", ""))
            passed = scores.get("overall", 0) >= case.get("min_score", 3.5)
            results.append({
                "query": case["query"],
                "expected_min_score": case.get("min_score", 3.5),
                "actual_score": scores.get("overall", 0),
                "passed": passed
            })

        passed_count = sum(1 for r in results if r["passed"])
        return {
            "total": len(results),
            "passed": passed_count,
            "failed": len(results) - passed_count,
            "pass_rate": passed_count / len(results) if results else 0,
            "details": results
        }
```

核心思想：维护一组"黄金测试集"——已知正确的输入输出对。每次修改代码或 prompt 后跑一遍，确保没有退化。

## 四、哪些测试值得先做

如果你时间有限（通常都有限），不要试图一次性搭全四层测试。按优先级来：

**第一优先级：回归测试的黄金集。** 把你遇到过的所有 bad case 攒成一个 JSON 文件，每次改 prompt 或代码之后跑一遍。这是性价比最高的测试——10 分钟搭完，能挡住 80% 的退化问题。

**第二优先级：工具选择的组件测试。** 如果你的 Agent 有 5 个以上的工具，工具选错是最常见的翻车场景。写 20 条测试用例（"用户说 X 时应该选工具 Y"），跑出一个准确率数字。低于 90% 就去优化工具描述。

**第三优先级：行为测试。** 等你的 Agent 已经稳定运行一段时间之后，再考虑用 LLM-as-Judge 做系统性的行为评估。这个成本高（每跑一次都要调 LLM），适合每周或每次大改后做一次。

单元测试对传统函数该写还是写，但对 Agent 来说优先级反而最低——因为 Agent 的核心问题不是"函数算错了"，而是"它选错了工具"或者"它编了个不存在的功能"。这些问题传统单元测试覆盖不了。
