## 一、裸写智能体的痛点

先回忆一下第 1 篇我们裸写智能体的样子：

```python
# 第 1 篇的做法
while True:
    response = client.chat.completions.create(model=MODEL_NAME, messages=messages, tools=tools)
    msg = response.choices[0].message
    if msg.tool_calls:
        for call in msg.tool_calls:
            name = call.function.name
            args = json.loads(call.function.arguments)
            if name == "get_weather":
                result = get_weather(**args)
            elif name == "search_news":
                result = search_news(**args)
            elif name == "calculate":
                result = calculate(**args)
            # 每加一个工具就多一个 elif...
    else:
        break
```

能跑，但有三个问题越来越严重：

**问题 1：工具管理是硬编码的**。每加一个工具就改一次 if-elif，10 个工具就 10 个分支，100 个工具代码直接爆炸。

**问题 2：没有中间件机制**。想加日志？改主循环。想加重试？改主循环。想加限流？还是改主循环。所有横切逻辑都往主循环里塞，代码变成一坨面条。

**问题 3：不可组合**。想让两个 Agent 协作？自己手写消息传递。想在 Agent 外面套一层 Planning？又是一堆胶水代码。

框架就是来解决这些问题的。下面我们一步步把这些痛点干掉。

## 二、从零开始：V1 —— 工具注册表

第一步解决工具管理问题。核心思路：**用一个注册表取代 if-elif**。

```python
import os
import json
from typing import Callable, Any
from openai import OpenAI

os.environ["OPENAI_API_KEY"] = "你的API Key"
client = OpenAI(
    base_url="https://api.deepseek.com",
    api_key=os.getenv("DEEPSEEK_API_KEY")
)
MODEL_NAME = "deepseek-chat"


class ToolRegistry:
    """工具注册表：用装饰器注册工具，自动生成 schema，按名字分发调用"""

    def __init__(self):
        self._tools: dict[str, dict] = {}

    def register(self, description: str, parameters: dict):
        """装饰器：注册一个工具"""
        def decorator(func: Callable) -> Callable:
            self._tools[func.__name__] = {
                "function": func,
                "schema": {
                    "type": "function",
                    "function": {
                        "name": func.__name__,
                        "description": description,
                        "parameters": parameters
                    }
                }
            }
            return func
        return decorator

    def get_schemas(self) -> list[dict]:
        """获取所有工具的 schema（传给 OpenAI API）"""
        return [t["schema"] for t in self._tools.values()]

    def call(self, name: str, arguments: dict) -> Any:
        """按名字调用工具"""
        if name not in self._tools:
            return f"错误：未知工具 '{name}'"
        return self._tools[name]["function"](**arguments)

    def list_tools(self) -> list[str]:
        return list(self._tools.keys())


# 创建注册表并注册工具
registry = ToolRegistry()


@registry.register(
    description="查询指定城市的天气",
    parameters={
        "type": "object",
        "properties": {
            "city": {"type": "string", "description": "城市名称"}
        },
        "required": ["city"]
    }
)
def get_weather(city: str) -> str:
    """模拟天气 API"""
    weather_data = {"北京": "晴，28°C", "上海": "多云，25°C", "深圳": "雷阵雨，31°C"}
    return weather_data.get(city, f"{city}：暂无数据")


@registry.register(
    description="搜索新闻",
    parameters={
        "type": "object",
        "properties": {
            "keyword": {"type": "string", "description": "搜索关键词"}
        },
        "required": ["keyword"]
    }
)
def search_news(keyword: str) -> str:
    """模拟新闻搜索"""
    return f"搜索到 3 条关于 '{keyword}' 的新闻：1. {keyword}最新进展 2. {keyword}行业分析 3. {keyword}未来趋势"


@registry.register(
    description="数学计算",
    parameters={
        "type": "object",
        "properties": {
            "expression": {"type": "string", "description": "数学表达式"}
        },
        "required": ["expression"]
    }
)
def calculate(expression: str) -> str:
    """安全的数学计算"""
    allowed = set("0123456789+-*/.(). ")
    if not all(c in allowed for c in expression):
        return "错误：只支持基本数学运算"
    try:
        return str(eval(expression))
    except Exception as e:
        return f"计算错误：{e}"


print(f"已注册工具：{registry.list_tools()}")
```

现在加一个工具只需要加一个 `@registry.register` 装饰器，不用改任何其他代码。这就是**注册表模式**——框架的第一块基石。

## 三、V2 —— Agent 基类

有了工具注册表，接下来把 Agent 本身抽象出来。核心是一个基类，封装了"循环调用 LLM + 分发工具"的逻辑：

```python
class BaseAgent:
    """智能体基类：封装核心循环"""

    def __init__(
        self,
        name: str,
        system_prompt: str,
        tool_registry: ToolRegistry | None = None,
        max_iterations: int = 10
    ):
        self.name = name
        self.system_prompt = system_prompt
        self.registry = tool_registry or ToolRegistry()
        self.max_iterations = max_iterations
        self.history: list[dict] = [{"role": "system", "content": system_prompt}]

    def run(self, user_input: str) -> str:
        """运行智能体：核心循环"""
        self.history.append({"role": "user", "content": user_input})
        tools = self.registry.get_schemas() or None

        for iteration in range(self.max_iterations):
            response = client.chat.completions.create(
                model=MODEL_NAME,
                messages=self.history,
                tools=tools
            )
            msg = response.choices[0].message

            if not msg.tool_calls:
                self.history.append({"role": "assistant", "content": msg.content})
                return msg.content

            # 有工具调用
            self.history.append({
                "role": "assistant",
                "content": msg.content,
                "tool_calls": [
                    {
                        "id": tc.id,
                        "type": "function",
                        "function": {"name": tc.function.name, "arguments": tc.function.arguments}
                    }
                    for tc in msg.tool_calls
                ]
            })

            for tool_call in msg.tool_calls:
                name = tool_call.function.name
                args = json.loads(tool_call.function.arguments)
                result = self.registry.call(name, args)
                self.history.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": str(result)
                })

        return "达到最大迭代次数，任务未完成。"

    def reset(self):
        """重置对话历史"""
        self.history = [{"role": "system", "content": self.system_prompt}]


# 用法：3 行代码创建一个能用工具的智能体
agent = BaseAgent(
    name="助手",
    system_prompt="你是一个全能助手，可以查天气、搜新闻、做计算。",
    tool_registry=registry
)

answer = agent.run("北京今天天气怎么样？另外帮我算一下 1024 * 768")
print(f"回答：{answer}")
```

对比一下裸写时代：以前加工具要改主循环，现在只要往 `registry` 里注册就行；以前每个 Agent 都要写一遍循环，现在继承 `BaseAgent` 就有了。

## 四、V3 —— 中间件系统

真正让框架变得强大的是中间件。日志、重试、限流、缓存——这些**横切关注点**不应该塞在主循环里，而应该以中间件的形式"插入"到流程中。

### 4.1 中间件协议

```python
import time
from typing import Protocol


class Middleware(Protocol):
    """中间件协议：所有中间件都要实现这两个方法"""

    def before_llm_call(self, messages: list[dict]) -> list[dict]:
        """LLM 调用前：可以修改 messages"""
        ...

    def after_llm_call(self, response: Any) -> Any:
        """LLM 调用后：可以修改或记录 response"""
        ...


class LoggingMiddleware:
    """日志中间件：记录每次 LLM 调用"""

    def before_llm_call(self, messages: list[dict]) -> list[dict]:
        msg_count = len(messages)
        last_msg = messages[-1]["content"][:50] if messages else ""
        print(f"  [LOG] → LLM 调用，{msg_count} 条消息，最后一条：{last_msg}...")
        self._start_time = time.time()
        return messages

    def after_llm_call(self, response: Any) -> Any:
        elapsed = time.time() - self._start_time
        print(f"  [LOG] ← LLM 响应，耗时 {elapsed:.2f}s")
        return response


class RetryMiddleware:
    """重试中间件：LLM 调用失败时自动重试"""

    def __init__(self, max_retries: int = 3, delay: float = 1.0):
        self.max_retries = max_retries
        self.delay = delay
        self._retry_count = 0

    def before_llm_call(self, messages: list[dict]) -> list[dict]:
        self._retry_count = 0
        return messages

    def after_llm_call(self, response: Any) -> Any:
        return response


class TokenCountMiddleware:
    """Token 计费中间件：统计每次调用的 token 消耗"""

    def __init__(self):
        self.total_prompt_tokens = 0
        self.total_completion_tokens = 0
        self.call_count = 0

    def before_llm_call(self, messages: list[dict]) -> list[dict]:
        return messages

    def after_llm_call(self, response: Any) -> Any:
        if hasattr(response, "usage") and response.usage:
            self.total_prompt_tokens += response.usage.prompt_tokens
            self.total_completion_tokens += response.usage.completion_tokens
            self.call_count += 1
        return response

    def report(self) -> str:
        total = self.total_prompt_tokens + self.total_completion_tokens
        return (
            f"API 调用 {self.call_count} 次 | "
            f"输入 {self.total_prompt_tokens} tokens | "
            f"输出 {self.total_completion_tokens} tokens | "
            f"总计 {total} tokens"
        )
```

### 4.2 支持中间件的 Agent

```python
class MiddlewareAgent(BaseAgent):
    """支持中间件的智能体"""

    def __init__(self, middlewares: list | None = None, **kwargs):
        super().__init__(**kwargs)
        self.middlewares = middlewares or []

    def _call_llm(self, messages: list[dict], tools: list[dict] | None) -> Any:
        """被中间件包裹的 LLM 调用"""
        # before 钩子
        current_messages = messages
        for mw in self.middlewares:
            current_messages = mw.before_llm_call(current_messages)

        # 实际调用
        response = client.chat.completions.create(
            model=MODEL_NAME,
            messages=current_messages,
            tools=tools
        )

        # after 钩子
        for mw in reversed(self.middlewares):
            response = mw.after_llm_call(response)

        return response

    def run(self, user_input: str) -> str:
        """重写 run 方法，使用中间件包裹的 LLM 调用"""
        self.history.append({"role": "user", "content": user_input})
        tools = self.registry.get_schemas() or None

        for iteration in range(self.max_iterations):
            response = self._call_llm(self.history, tools)
            msg = response.choices[0].message

            if not msg.tool_calls:
                self.history.append({"role": "assistant", "content": msg.content})
                return msg.content

            self.history.append({
                "role": "assistant",
                "content": msg.content,
                "tool_calls": [
                    {
                        "id": tc.id,
                        "type": "function",
                        "function": {"name": tc.function.name, "arguments": tc.function.arguments}
                    }
                    for tc in msg.tool_calls
                ]
            })

            for tool_call in msg.tool_calls:
                name = tool_call.function.name
                args = json.loads(tool_call.function.arguments)
                result = self.registry.call(name, args)
                self.history.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": str(result)
                })

        return "达到最大迭代次数，任务未完成。"


# 组装：Agent + 中间件
token_counter = TokenCountMiddleware()

agent = MiddlewareAgent(
    name="带中间件的助手",
    system_prompt="你是一个全能助手。",
    tool_registry=registry,
    middlewares=[
        LoggingMiddleware(),
        token_counter,
    ]
)

answer = agent.run("深圳天气怎么样？")
print(f"\n回答：{answer}")
print(f"统计：{token_counter.report()}")
```

看到没？加日志、加 Token 统计，一行代码都没改主循环——只是在创建 Agent 时多传了两个中间件。**这就是框架的威力：把"怎么做"和"做什么"分开**。

## 五、V4 —— Agent 组合与编排

单个 Agent 能力有限，真正有用的是**多个 Agent 组合**。框架需要提供组合机制。

### 5.1 顺序编排（Pipeline）

```python
class Pipeline:
    """顺序编排：Agent A 的输出作为 Agent B 的输入"""

    def __init__(self, agents: list[BaseAgent]):
        self.agents = agents

    def run(self, initial_input: str) -> str:
        current_input = initial_input
        results: list[str] = []

        for i, agent in enumerate(self.agents):
            agent.reset()
            output = agent.run(current_input)
            results.append(output)
            # 下一个 Agent 的输入 = 上一个的输出
            current_input = f"基于以下内容继续工作：\n\n{output}"
            print(f"  [{agent.name}] 完成")

        return results[-1]


# 三个角色串成流水线
researcher = BaseAgent(
    name="研究员",
    system_prompt="你是技术研究员。收到一个主题后，列出 3-5 个关键要点。只输出要点，不要废话。"
)

analyst = BaseAgent(
    name="分析师",
    system_prompt="你是技术分析师。收到一组要点后，对每个要点给出优劣分析。简洁明了。"
)

writer = BaseAgent(
    name="写手",
    system_prompt="你是技术写手。收到分析后，整理成一篇简短的对比报告，300 字以内。"
)

pipeline = Pipeline([researcher, analyst, writer])
report = pipeline.run("对比 Python 的 FastAPI、Django、Flask 三个 Web 框架")
print(f"\n最终报告：\n{report}")
```

### 5.2 路由编排（Router）

有时候不是所有 Agent 都要跑，而是根据问题类型路由到合适的 Agent：

```python
class Router:
    """路由编排：根据问题类型分发到不同的 Agent"""

    def __init__(self, routes: dict[str, BaseAgent], default: BaseAgent | None = None):
        self.routes = routes
        self.default = default

    def run(self, user_input: str) -> str:
        # 用 LLM 判断应该路由到哪个 Agent
        route_names = ", ".join(self.routes.keys())
        response = client.chat.completions.create(
            model=MODEL_NAME,
            messages=[{
                "role": "user",
                "content": (
                    f"判断以下用户输入应该由哪个专家处理。\n"
                    f"可选专家：{route_names}\n"
                    f"用户输入：{user_input}\n\n"
                    f"只返回专家名称，不要其他内容。"
                )
            }],
            temperature=0
        )

        chosen = response.choices[0].message.content.strip()
        agent = self.routes.get(chosen, self.default)
        if agent is None:
            return f"无法确定路由目标：{chosen}"

        print(f"  [路由] → {chosen}")
        agent.reset()
        return agent.run(user_input)


# 三个专精 Agent
code_agent = BaseAgent(
    name="代码专家",
    system_prompt="你是 Python 代码专家，擅长写代码和解释代码。",
    tool_registry=registry
)

devops_agent = BaseAgent(
    name="运维专家",
    system_prompt="你是运维专家，擅长部署、监控和性能调优。"
)

general_agent = BaseAgent(
    name="通用助手",
    system_prompt="你是通用助手，什么都能聊。"
)

router = Router(
    routes={
        "代码专家": code_agent,
        "运维专家": devops_agent,
        "通用助手": general_agent,
    },
    default=general_agent
)

# 不同问题自动路由到不同 Agent
for q in ["帮我写一个快排", "Nginx 怎么配反向代理？", "今天天气不错"]:
    print(f"\n用户：{q}")
    answer = router.run(q)
    print(f"回答：{answer[:100]}...")
```

## 六、完整框架：把所有东西组装起来

把前面四个版本的能力合并，形成一个完整的迷你框架：

```python
class MiniAgentFramework:
    """
    迷你智能体框架
    核心组件：
    1. ToolRegistry —— 工具注册与分发
    2. BaseAgent / MiddlewareAgent —— 智能体基类
    3. Middleware —— 中间件（日志、Token 统计等）
    4. Pipeline / Router —— 多 Agent 编排
    """

    def __init__(self):
        self.registry = ToolRegistry()
        self.agents: dict[str, BaseAgent] = {}

    def tool(self, description: str, parameters: dict):
        """注册工具的快捷方式"""
        return self.registry.register(description, parameters)

    def create_agent(
        self,
        name: str,
        system_prompt: str,
        middlewares: list | None = None,
        use_tools: bool = True
    ) -> MiddlewareAgent:
        """创建并注册一个 Agent"""
        agent = MiddlewareAgent(
            name=name,
            system_prompt=system_prompt,
            tool_registry=self.registry if use_tools else ToolRegistry(),
            middlewares=middlewares or []
        )
        self.agents[name] = agent
        return agent

    def pipeline(self, agent_names: list[str]) -> Pipeline:
        """创建顺序编排"""
        agents = [self.agents[n] for n in agent_names]
        return Pipeline(agents)

    def router(self, route_map: dict[str, str], default: str | None = None) -> Router:
        """创建路由编排"""
        routes = {k: self.agents[v] for k, v in route_map.items()}
        default_agent = self.agents[default] if default else None
        return Router(routes, default_agent)


# —— 使用示例 ——

framework = MiniAgentFramework()

# 注册工具
@framework.tool(
    description="查询天气",
    parameters={"type": "object", "properties": {"city": {"type": "string"}}, "required": ["city"]}
)
def weather(city: str) -> str:
    return f"{city}：晴，28°C"

@framework.tool(
    description="数学计算",
    parameters={"type": "object", "properties": {"expr": {"type": "string"}}, "required": ["expr"]}
)
def calc(expr: str) -> str:
    return str(eval(expr))

# 创建 Agent
token_mw = TokenCountMiddleware()

framework.create_agent(
    name="通用助手",
    system_prompt="你是一个全能助手，能查天气、做计算。",
    middlewares=[LoggingMiddleware(), token_mw]
)

framework.create_agent(
    name="研究员",
    system_prompt="你是技术研究员，列出关键要点。",
    use_tools=False
)

framework.create_agent(
    name="写手",
    system_prompt="你是技术写手，整理成简短报告。",
    use_tools=False
)

# 方式 1：直接用单个 Agent
agent = framework.agents["通用助手"]
print(agent.run("北京天气怎么样？算一下 256*512"))

# 方式 2：流水线编排
pipe = framework.pipeline(["研究员", "写手"])
print(pipe.run("Python 异步编程的最佳实践"))

print(f"\nToken 统计：{token_mw.report()}")
```

## 七、主流框架做了什么：对号入座

现在回过头看那四个框架，你会发现它们的核心抽象和我们的迷你框架是一回事：

| 我们的框架 | CrewAI | AutoGen | LangGraph | MetaGPT |
|-----------|--------|---------|-----------|---------|
| `ToolRegistry` | `@tool` 装饰器 | `register_function` | `ToolNode` | `Action` |
| `BaseAgent` | `Agent` 类 | `AssistantAgent` | 图中的节点函数 | `Role` 类 |
| `Middleware` | 无（较弱） | `register_hook` | 图的 `before`/`after` | `Environment` |
| `Pipeline` | `Crew(process=sequential)` | `GroupChat` | `StateGraph` 线性边 | `Team.run()` |
| `Router` | `Crew(process=hierarchical)` | `speaker_selection` | `conditional_edge` | `Role` 路由 |

差别在哪？

- **CrewAI**：在我们的基础上加了角色扮演（backstory）、任务描述（expected_output）、自动委派（delegation）。对"人设驱动"的场景很友好。

- **AutoGen**：把 Agent 间通信做成了对话——每个 Agent 有独立的聊天记录，通过 GroupChat 管理多人对话。适合需要"讨论"的场景。

- **LangGraph**：把编排做成了图——节点是函数，边是条件，支持循环和并行。状态管理最灵活，但上手成本也最高。

- **MetaGPT**：在我们的基础上加了"标准化输出"——每个角色的输出格式是固定的（PRD、架构文档、代码），下游角色直接消费上游的结构化产物。适合代码生成流水线。

## 八、什么时候该用框架，什么时候该裸写？

| 场景 | 建议 |
|------|------|
| PoC / Demo / 学习 | **裸写**。理解原理最重要，框架反而添乱 |
| 单 Agent + 几个工具 | **裸写**。20 行代码搞定的事不需要框架 |
| 多 Agent 协作 | **用框架**。消息传递、调度逻辑自己写会疯 |
| 生产环境 | **用框架**。日志、重试、监控、可观测性，框架都帮你想好了 |
| 定制化极强 | **造框架**。主流框架不满足需求时，基于本文的思路自己搭 |

一句话总结：**简单的事裸写，复杂的事用框架，框架不够用就造框架**。

## 九、总结

| 版本 | 解决什么问题 | 核心设计 |
|------|-------------|----------|
| V1 工具注册表 | 硬编码 if-elif | 装饰器 + 注册表模式 |
| V2 Agent 基类 | 重复的循环逻辑 | 继承 + 模板方法 |
| V3 中间件 | 横切关注点耦合 | 洋葱模型 + 钩子函数 |
| V4 编排 | 多 Agent 组合 | Pipeline + Router 模式 |

框架不是魔法，就是把反复出现的模式抽象出来，让你少写重复代码。理解了这些抽象，你不仅能用好框架，还能在框架不够用时自己扩展。
