## 一、开头引入

之前写多智能体协作那篇，咱们纯手写了主从、辩论、流水线三种模式。能跑，但代码都是自己造的轮子——消息传递、任务分配、错误处理全得自己写。如果生产项目要搞多智能体，总不能什么都从头来。这四个框架就是来解决这个问题的。

说白了，它们帮你把「多个 Agent 怎么协作」这件事抽象好了，你只需要定义角色和任务，框架负责调度、传消息、处理异常。今天咱们用同一个任务，把四个框架都跑一遍，看看谁更适合你。注意：下面代码里的 API 可能随版本更新，以各框架官方文档为准。

---

## 二、先搞清楚这四个框架的定位

选框架之前，得先知道它们各自擅长啥。不然就像去超市买酱油，结果抱了一瓶醋回来。

**CrewAI**：最简单好用的多智能体框架。核心概念是 Agent + Task + Crew，就像组一个项目组——你招几个人（Agent），给他们分配活（Task），然后让他们按顺序或层级干完（Crew）。API 设计得很直白，10 分钟能跑通。

**AutoGen（微软出品）**：以「对话」为核心。Agent 之间通过聊天协作，你一句我一句，适合需要来回讨论、辩论、推理的场景。比如让一个 Agent 提方案，另一个挑毛病，第三个总结——这种多轮交互是 AutoGen 的强项。

**LangGraph（LangChain 出品）**：基于图的工作流编排。把智能体流程建模成状态机，节点是函数，边是条件跳转。适合需要条件分支、循环、并行执行的复杂流程。想画流程图的那种需求，用 LangGraph 最顺手。

**MetaGPT**：模拟软件公司的多智能体框架。内置产品经理、架构师、工程师等角色，你给一句需求，它自动分工、写 PRD、画架构图、生成代码。适合代码生成，但比较重，定制化难度高。

这四个框架的底层思路不一样：CrewAI 和 MetaGPT 偏「任务驱动」，Agent 按既定流程干活；AutoGen 和 LangGraph 偏「流程驱动」，前者靠对话推进，后者靠图结构推进。理解这一点，选型时就不会迷糊。

## 三、统一任务：同一道题，四种解法

为了公平对比，咱们用同一个任务跑四个框架：

**任务**：帮我调研 Python Web 框架（Django / Flask / FastAPI），输出一篇对比分析报告。

这个任务需要：搜集信息 → 分析对比 → 撰写成文。三个步骤，正好考验框架的协作能力。选这个任务的原因很简单：它不涉及代码执行（省掉工具调用的复杂度），又能体现多角色协作——研究员、分析师、写手各司其职，信息在它们之间流转。

## 四、CrewAI 实现

CrewAI 的思路最直观：定义 3 个 Agent（研究员、分析师、写手），3 个 Task，1 个 Crew，然后 kickoff。

```bash
pip install crewai crewai-tools
```

```python
import os
os.environ["OPENAI_API_KEY"] = "your-api-key"  # 或从环境变量读取

from crewai import Agent, Task, Crew, Process

# 定义三个 Agent
researcher = Agent(
    role="技术研究员",
    goal="搜集 Django、Flask、FastAPI 的官方文档和社区评价",
    backstory="你是资深 Python 开发者，擅长技术调研",
    allow_delegation=False,
    verbose=True
)

analyst = Agent(
    role="技术分析师",
    goal="对比三个框架的优缺点、适用场景和性能表现",
    backstory="你是架构师，擅长技术选型分析",
    allow_delegation=False,
    verbose=True
)

writer = Agent(
    role="技术写手",
    goal="将分析结果整理成结构清晰的对比报告",
    backstory="你是技术文档作者，擅长把复杂信息讲清楚",
    allow_delegation=False,
    verbose=True
)

# 定义三个 Task
research_task = Task(
    description="调研 Django、Flask、FastAPI 三个 Python Web 框架，包括：特性、生态、社区活跃度",
    expected_output="一份调研笔记，包含三个框架的核心信息",
    agent=researcher
)

analysis_task = Task(
    description="基于调研笔记，对比三个框架的优缺点、适用场景和性能表现",
    expected_output="一份对比分析表格和结论",
    agent=analyst,
    context=[research_task]
)

write_task = Task(
    description="根据对比分析，撰写一篇完整的 Python Web 框架对比报告，适合技术选型参考",
    expected_output="一篇 1500 字左右的对比分析报告，含结论和推荐",
    agent=writer,
    context=[research_task, analysis_task]
)

# 创建 Crew，按顺序执行
crew = Crew(
    agents=[researcher, analyst, writer],
    tasks=[research_task, analysis_task, write_task],
    process=Process.sequential
)

# 执行
result = crew.kickoff()
print(result)
```

**模拟运行效果：**

```
[Researcher] 正在调研 Django、Flask、FastAPI...
[Researcher] 调研完成，输出：Django 全栈、ORM 内置；Flask 轻量、灵活；FastAPI 异步、高性能...
[Analyst] 基于调研笔记进行对比分析...
[Analyst] 分析完成：Django 适合大型项目，Flask 适合快速原型，FastAPI 适合 API 服务...
[Writer] 撰写对比报告...
[Writer] 报告完成。

--- 最终输出 ---
# Python Web 框架对比分析报告

## 一、综述
Django、Flask、FastAPI 是 Python 生态中最主流的三个 Web 框架...

## 二、详细对比
| 维度 | Django | Flask | FastAPI |
|------|--------|-------|---------|
| 定位 | 全栈框架 | 微框架 | 异步 API 框架 |
...
```

CrewAI 的核心优势：API 最简洁，`Agent` + `Task` + `Crew` 三板斧，10 分钟能跑通。适合快速原型和通用多 Agent 场景。`context` 参数让下游 Task 能拿到上游的输出，这种依赖关系写得很清楚。缺点嘛，流程固定，想加条件分支（比如「分析结果太短就重做」）就得自己绕一下。

## 五、AutoGen 实现

AutoGen 的玩法不一样：Agent 之间通过聊天协作。你定义一个 SelectorGroupChat（或 RoundRobinGroupChat），塞几个 AssistantAgent，框架会用 LLM 选「下一个谁说话」，然后让它们自己聊。

```bash
pip install "autogen-agentchat" "autogen-ext[openai]"
```

```python
import os
import asyncio
os.environ["OPENAI_API_KEY"] = "your-api-key"

from autogen_agentchat.agents import AssistantAgent
from autogen_agentchat.teams import SelectorGroupChat
from autogen_agentchat.conditions import TextMentionTermination, MaxMessageTermination
from autogen_agentchat.ui import Console
from autogen_ext.models.openai import OpenAIChatCompletionClient

model_client = OpenAIChatCompletionClient(model="gpt-4o-mini")

# 研究员：负责搜集信息
researcher = AssistantAgent(
    name="Researcher",
    description="负责搜集技术信息，输出调研笔记",
    model_client=model_client,
    system_message="你是技术研究员。搜集 Django、Flask、FastAPI 的信息，用 bullet points 输出调研结果。完成后说 DONE。"
)

# 分析师：负责对比分析
analyst = AssistantAgent(
    name="Analyst",
    description="负责对比分析，输出结论",
    model_client=model_client,
    system_message="你是技术分析师。基于 Researcher 的调研结果，对比三个框架的优缺点和适用场景。完成后说 DONE。"
)

# 写手：负责撰写报告
writer = AssistantAgent(
    name="Writer",
    description="负责撰写最终报告",
    model_client=model_client,
    system_message="你是技术写手。根据调研和分析结果，撰写完整对比报告。完成后说 TERMINATE。"
)

# 终止条件：写手说 TERMINATE 或最多 15 条消息
termination = TextMentionTermination("TERMINATE") | MaxMessageTermination(max_messages=15)

# 创建 GroupChat 团队
team = SelectorGroupChat(
    [researcher, analyst, writer],
    model_client=model_client,
    termination_condition=termination,
)

# 执行（需在 async 环境中运行）
async def main():
    await Console(team.run_stream(task="请调研 Python Web 框架（Django、Flask、FastAPI），输出一篇对比分析报告。"))

asyncio.run(main())
```

**模拟运行效果：**

```
[UserProxy] 请帮我调研 Python Web 框架（Django、Flask、FastAPI），输出一篇对比分析报告。

[Researcher] 正在调研...
- Django: 全栈框架，内置 ORM、Admin、认证...
- Flask: 微框架，轻量灵活，扩展丰富...
- FastAPI: 基于 Starlette，异步高性能...

[Analyst] 基于调研，对比分析如下：
- 选型建议：大型项目用 Django，快速原型用 Flask，API 服务用 FastAPI

[Writer] 撰写报告...
[Writer] 报告完成。以下是完整对比报告：...
```

AutoGen 的核心优势：对话式协作很自然，Agent 之间可以来回讨论、互相补充。适合需要多轮推理、辩论的场景。比如你要写一篇技术方案，可以让一个 Agent 先出初稿，另一个挑毛病，第三个根据反馈再改——这种「你来我往」在 AutoGen 里很自然。缺点是流程不可控，你不知道它什么时候停，得靠 `termination_condition` 或 `max_round` 兜底。

## 六、LangGraph 实现

LangGraph 把流程建模成图：节点是函数，边是条件跳转。适合需要条件分支、循环、并行执行的复杂工作流。

```bash
pip install langgraph langchain-openai
```

```python
import os
os.environ["OPENAI_API_KEY"] = "your-api-key"

from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage

# 定义状态
class ResearchState(TypedDict):
    research: str
    analysis: str
    report: str

# 初始化 LLM
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# 节点函数
def research_node(state: ResearchState) -> ResearchState:
    prompt = """调研 Django、Flask、FastAPI 三个 Python Web 框架，
    输出调研笔记，包含特性、生态、社区活跃度。"""
    response = llm.invoke([HumanMessage(content=prompt)])
    return {"research": response.content}

def analysis_node(state: ResearchState) -> ResearchState:
    prompt = f"""基于以下调研笔记，对比三个框架的优缺点和适用场景：
    {state['research']}
    输出对比分析表格和结论。"""
    response = llm.invoke([HumanMessage(content=prompt)])
    return {"analysis": response.content}

def write_node(state: ResearchState) -> ResearchState:
    prompt = f"""根据调研和分析结果，撰写一篇 Python Web 框架对比报告：
    调研：{state['research']}
    分析：{state['analysis']}
    输出 1500 字左右的完整报告，含结论和推荐。"""
    response = llm.invoke([HumanMessage(content=prompt)])
    return {"report": response.content}

# 构建图
workflow = StateGraph(ResearchState)
workflow.add_node("research", research_node)
workflow.add_node("analysis", analysis_node)
workflow.add_node("write", write_node)

workflow.add_edge(START, "research")
workflow.add_edge("research", "analysis")
workflow.add_edge("analysis", "write")
workflow.add_edge("write", END)

# 编译并执行
app = workflow.compile()
result = app.invoke({"research": "", "analysis": "", "report": ""})
print(result["report"])
```

**模拟运行效果：**

```
[research_node] 执行调研...
[analysis_node] 执行分析...
[write_node] 执行撰写...

--- 最终输出 ---
# Python Web 框架对比分析报告
...
```

LangGraph 的核心优势：流程可视化、有状态管理、适合复杂工作流。如果以后要加「分析失败则重试」「根据报告长度决定是否继续」等条件分支，改图就行，不用大动框架。`add_conditional_edges` 可以按状态决定下一步走哪条边，这是 CrewAI 和 AutoGen 不容易做到的。代价是学习曲线陡一点，你得先理解「图」的思维。

## 七、MetaGPT 实现

MetaGPT 是「开箱即用的软件开发团队」。你给一句需求，它自动分工：产品经理写 PRD，架构师画架构图，工程师写代码。咱们这个调研任务用 MetaGPT 有点「杀鸡用牛刀」，但可以看看它的用法。

```bash
pip install metagpt
```

```python
import os
os.environ["OPENAI_API_KEY"] = "your-api-key"

from metagpt.software_company import generate_repo

# 一句话需求，自动生成完整项目
generate_repo(
    idea="Create a Python script that researches Django, Flask, FastAPI and outputs a comparison report",
    investment=2.0,  # 预算（控制生成深度）
    n_round=3,       # 迭代轮数
    code_review=True,
    run_tests=False
)
```

MetaGPT 内置了 SoftwareCompany 团队，直接给需求它自动分工。核心优势：开箱即用的软件开发团队，适合代码生成。局限：比较重，定制化难度高，想改角色或流程得啃源码。`investment` 参数控制「预算」，数值越大生成越深；`n_round` 控制迭代轮数。如果你要做的是「给需求出代码」这类事，MetaGPT 很合适；如果只是调研、写报告，用前面三个框架更轻量。

## 八、四框架正面对比表

| 维度 | CrewAI | AutoGen | LangGraph | MetaGPT |
|------|--------|---------|-----------|---------|

| 生产就绪度 | 较好 | 好 | 好 | 一般 |

## 九、选型决策树

- **想快速跑通多 Agent** → CrewAI
- **需要 Agent 来回讨论** → AutoGen
- **需要复杂流程（条件分支 / 循环 / 并行）** → LangGraph
- **做代码生成 / 软件开发 Agent** → MetaGPT
- **不确定** → 先用 CrewAI，够用就别换

## 十、写在最后

框架选型的核心不是哪个最强，而是哪个最适合你的场景。如果你的需求很简单，上一篇文章里手写的方案可能就够了。框架的价值在于帮你省掉底层的脏活，让你专注业务逻辑。

四个框架都跑一遍之后，你大概能感觉到：CrewAI 像「项目管理软件」，任务拆好、人派好，按流程走；AutoGen 像「群聊」，大家你一言我一语把事聊完；LangGraph 像「流程图工具」，节点和边画清楚，执行按图来；MetaGPT 像「外包团队」，需求丢进去，代码交出来。类比不一定精确，但能帮你快速建立直觉。

下一篇咱们单独深挖 LangGraph——因为它的「图工作流」思路确实比其他框架灵活一档。想搞复杂流程的，可以提前关注。
