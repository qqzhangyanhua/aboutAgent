## 一、智能体到底是个啥？

动手之前，先用 1 分钟对齐一下认知。

你平时用 ChatGPT，交互模式是这样的：

> 你问一句 → AI 回一句 → 完事。

但智能体不一样。你跟它说"帮我规划一下周末出游"，它会**自己**去查天气、**自己**去搜景点、**自己**算费用，全搞定了再给你一份完整方案。

区别在哪？就俩字——**自主**。

ChatGPT 是个问答机器，你不问它不动；智能体更像一个实习生，你给它一个模糊的需求，它自己拆任务、找工具、干活、交付。

用公式说就是：**智能体 = 大模型 + 工具 + 循环**。大模型负责想，工具负责干，循环让这两件事不断交替，直到任务搞定。

---

## 二、拆开看：智能体里面有啥？

不管外面套了多花哨的壳，一个智能体拆到底就三样东西：

**大脑（LLM）** — GPT、Claude、Gemini、DeepSeek……随便哪个大模型都行。它的作用是"理解需求"和"做决策"，但注意，大模型本身不能上网，不能查数据库，它只能动嘴皮子。

**工具（Tools）** — 既然大模型只能动嘴，那干活就得靠工具。查天气的 API、计算器函数、搜索接口，这些都算工具。大模型决定"我需要查北京天气"，然后由我们的代码去实际调用天气 API。

**记忆（Memory）** — 说白了就是一个列表，把之前的对话、调过的工具、拿到的结果全存下来。每次调用大模型的时候把这个列表传过去，大模型才知道前面干了啥，不会重复劳动或者前后矛盾。

### 3.2 定义工具箱

先给智能体装上四个工具——查天气、算数学、搜新闻、查汇率。后面三个版本共用这一套：

```python
# ===== 工具函数（模拟真实API） =====

def get_weather(city: str) -> str:
    """模拟查天气"""
    fake_weather = {
        "北京": "晴，25°C，微风，湿度40%",
        "上海": "多云，22°C，东南风3级，湿度65%",
        "广州": "雷阵雨，30°C，闷热，湿度85%",
        "深圳": "阴转小雨，28°C，湿度78%",
        "杭州": "晴，24°C，西北风2级，湿度50%",
    }
    return fake_weather.get(city, f"{city}：暂无天气数据")

def calculate(expression: str) -> str:
    """计算数学表达式"""
    try:
        return str(eval(expression))
    except Exception as e:
        return f"计算出错: {e}"

def search_news(keyword: str) -> str:
    """模拟搜索新闻"""
    fake_news = {
        "AI": "【最新】OpenAI发布GPT-5，推理能力大幅提升；Google推出Gemini 2.5 Pro；"
              "国内多家大模型厂商宣布降价，AI应用开发成本持续下降。",
        "天气": "【预警】未来三天华南地区将迎来强降雨，气象台发布暴雨黄色预警。",
        "股市": "【财经】A股三大指数集体高开，AI概念股领涨，成交额突破万亿。",
    }
    for key, news in fake_news.items():
        if key in keyword:
            return news
    return f"未找到关于'{keyword}'的相关新闻"

def get_exchange_rate(from_currency: str, to_currency: str) -> str:
    """模拟查汇率"""
    rates = {
        ("USD", "CNY"): 7.24,
        ("EUR", "CNY"): 7.86,
        ("JPY", "CNY"): 0.048,
        ("GBP", "CNY"): 9.15,
        ("CNY", "USD"): 0.138,
    }
    rate = rates.get((from_currency.upper(), to_currency.upper()))
    if rate:
        return f"1 {from_currency.upper()} = {rate} {to_currency.upper()}"
    return f"暂不支持 {from_currency} → {to_currency} 的汇率查询"
```

```python
        }
    },

        "type": "function",
        "function": {
            "name": "search_news",
            "description": "根据关键词搜索最新新闻资讯",
            "parameters": {
                "type": "object",
                "properties": {
                    "keyword": {"type": "string", "description": "搜索关键词，如 AI、天气、股市"}

                "required": ["keyword"]
            }
        }

    {
        "type": "function",

# ===== 工具执行器 =====

available_tools = {
    "get_weather": get_weather,
    "calculate": calculate,
    "search_news": search_news,
    "get_exchange_rate": get_exchange_rate,
}

def execute_tool(tool_name: str, arguments: dict) -> str:
    if tool_name in available_tools:
        return available_tools[tool_name](**arguments)
    return f"未知工具: {tool_name}"
```

## 四、V1：最简版 — 能干活就行

V1 的逻辑特别直白：大模型说要用工具 → 我们执行工具 → 把结果喂回去 → 大模型继续想。如此循环，直到它说"我不需要工具了"为止。

```python
def run_agent_v1(user_input: str):
    """V1 基础版：简单的工具调用循环"""
    print(f"\n{'='*60}")
    print(f"  V1 基础版智能体")
    print(f"  用户: {user_input}")
    print(f"{'='*60}")

    messages = [
        {"role": "system", "content": "你是一个有用的AI助手，可以查天气、算数学、搜新闻、查汇率。"},
        {"role": "user", "content": user_input}
    ]

    for i in range(10):
        print(f"\n--- 第 {i+1} 轮 ---")
        response = client.chat.completions.create(
            model="gpt-4o-mini", messages=messages, tools=tools
        )
        msg = response.choices[0].message

        if not msg.tool_calls:
            print(f"\n✅ 最终回答: {msg.content}")
            return msg.content

        messages.append(msg)
        for tool_call in msg.tool_calls:
            name = tool_call.function.name
            args = json.loads(tool_call.function.arguments)
            print(f"  🔧 调用工具: {name}({args})")
            result = execute_tool(name, args)
            print(f"  📋 工具返回: {result}")
            messages.append({
                "role": "tool",
                "tool_call_id": tool_call.id,
                "content": result
            })

    return "达到最大轮次"
```

这就引出了 V2。

## 五、V2：ReAct 版 — 强制"先想后做"

### ReAct 是啥？

这个词来自 2023 年的一篇论文，全称 **Re**asoning + **Act**ing，翻译过来就是"边推理边行动"。

落地到代码上就一句话：**要求大模型在每次调用工具之前，先把自己的思考过程说出来。**

```
V1（闷头干）：  用户提问 → 调用工具 → 返回结果
V2（先想再干）：用户提问 → 【我觉得应该先...】→ 调用工具 → 【拿到结果了，接下来...】→ 继续
```

为啥要多这一步？两个好处：一是出了错你能定位到具体哪步想歪了；二是强迫大模型"先想后做"，本身就能降低犯错概率——跟我们做数学题先列思路再算一样。

### 代码实现

V2 和 V1 的代码结构几乎一样，区别只在 system prompt——多加了一条规矩：每次回复之前必须先用 `【思考】` 说清楚你在干嘛：

```python
REACT_SYSTEM_PROMPT = """你是一个善于思考的AI助手，可以查天气、算数学、搜新闻、查汇率。

你必须严格按照 ReAct 模式工作。在每次回复中，你要：

1. 先用【思考】标签说明你的推理过程：当前掌握了什么信息、还缺什么、下一步打算做什么、为什么
2. 然后再决定是调用工具还是直接回答

格式示例：
【思考】用户想知道北京和上海哪个更热。我需要分别查询两个城市的天气数据，然后比较温度。先查北京的天气。

注意事项：
- 每一轮都要有【思考】，不能跳过
- 思考要体现逻辑推理，不是简单复述任务
- 如果工具返回的结果有异常或不够用，要在思考中说明如何调整策略
- 最终回答时也要先有一段总结性的【思考】，然后再给出结论
"""

def run_agent_v2(user_input: str):
    """V2 ReAct版：带显式推理链的智能体"""
    print(f"\n{'='*60}")
    print(f"  V2 ReAct 版智能体")
    print(f"  用户: {user_input}")
    print(f"{'='*60}")

    messages = [
        {"role": "system", "content": REACT_SYSTEM_PROMPT},
        {"role": "user", "content": user_input}
    ]

    for i in range(10):
        print(f"\n--- 第 {i+1} 轮 ---")
        response = client.chat.completions.create(
            model="gpt-4o-mini", messages=messages, tools=tools
        )
        msg = response.choices[0].message

        # 打印大模型的思考过程（ReAct的核心价值）
        if msg.content:
            print(f"\n💭 {msg.content}")

        if not msg.tool_calls:
            print(f"\n✅ 最终回答已包含在上方思考中")
            return msg.content

        messages.append(msg)
        for tool_call in msg.tool_calls:
            name = tool_call.function.name
            args = json.loads(tool_call.function.arguments)
            print(f"  🔧 调用工具: {name}({args})")
            result = execute_tool(name, args)
            print(f"  📋 工具返回: {result}")
            messages.append({
                "role": "tool",
                "tool_call_id": tool_call.id,
                "content": result
            })

    return "达到最大轮次"
```


留意第 4 轮——模型心算已经得出 3620 了，但它还是选择用计算工具再验一下。这不是多此一举，"先想后做"的习惯会让大模型在关键环节更谨慎。

## 六、V3：反思版 — 干完活会自己回头查

### 为啥还需要反思？

V2 已经会"边想边干"了，但有个盲区——**它从不回头检查自己的工作**。

你自己写代码或者写文档，写完之后总会回头过一遍吧？看看有没有漏了什么、数字算对没有、逻辑通不通。这个"回头过一遍"的动作，就是反思（Reflection）。

落地到代码上，思路是这样的：先让智能体正常干活出一版初稿，然后换一个"质检员"角色去审查这个初稿——查完整性、查数据准确性、查有没有遗漏。如果质检不通过，就让它根据质检意见改一版再交。

```
V2：  收集信息 → 直接输出答案
V3：  收集信息 → 初版答案 → 质检审查 → 发现问题？→ 修正后输出
```

### 代码实现

在 V2 的基础上加两个新的 prompt：一个扮演"质检员"做审查，一个负责根据质检意见修正回答：

```python
REACT_SYSTEM_PROMPT_V3 = """你是一个善于深度思考的AI助手，可以查天气、算数学、搜新闻、查汇率。

你必须严格按照 ReAct 模式工作：
1. 每次回复先用【思考】说明推理过程
2. 然后决定是调用工具还是回答

注意：
- 每轮必须有【思考】
- 如果工具返回异常，在思考中说明调整策略
- 最终回答前要有总结性思考
"""

REFLECTION_PROMPT = """请你以"严格质检员"的身份，审视以下AI助手的回答。

用户的原始问题是：
{user_input}

AI助手的回答是：
{agent_answer}

请从以下维度检查，并输出审查报告：

1.【完整性】用户问了几个问题/子任务？是否每个都回答了？有没有遗漏？
2.【准确性】数据和计算是否正确？有没有张冠李戴或数字错误？
3.【逻辑性】推理过程是否合理？结论是否从证据中自然得出？有没有逻辑跳跃？
4.【实用性】给出的建议是否切实可行？有没有空话套话？
5.【潜在风险】有没有需要提醒用户但被忽略的重要信息？

最后给出：
- 综合评分：A（优秀）/ B（良好，有小瑕疵）/ C（及格，有明显遗漏）/ D（不及格，需要重做）
- 如果评分是 C 或 D，请明确指出需要补充或修正的内容
"""

REVISION_PROMPT = """你之前对用户的问题给出了一个回答，但质检发现了一些问题。

用户原始问题：
{user_input}

你之前的回答：
{agent_answer}

质检报告：
{reflection}

请根据质检报告的反馈，修正和完善你的回答。要求：
- 保留原回答中正确的部分
- 补充遗漏的内容
- 修正错误的数据或逻辑
- 输出一个更完整、更准确的最终回答
"""

def run_agent_v3(user_input: str):
    """V3 反思版：带自我审查和纠错的智能体"""
    print(f"\n{'='*60}")
    print(f"  V3 反思版智能体")
    print(f"  用户: {user_input}")
    print(f"{'='*60}")

    # ===== 阶段一：ReAct 执行，收集信息并生成初版回答 =====
    print(f"\n{'─'*40}")
    print("📝 阶段一：执行任务，生成初版回答")
    print(f"{'─'*40}")

    messages = [
        {"role": "system", "content": REACT_SYSTEM_PROMPT_V3},
        {"role": "user", "content": user_input}
    ]

    agent_answer = ""
    for i in range(10):
        print(f"\n--- 第 {i+1} 轮 ---")
        response = client.chat.completions.create(
            model="gpt-4o-mini", messages=messages, tools=tools
        )
        msg = response.choices[0].message

        if msg.content:
            print(f"\n💭 {msg.content}")

        if not msg.tool_calls:
            agent_answer = msg.content
            break

        messages.append(msg)
        for tool_call in msg.tool_calls:
            name = tool_call.function.name
            args = json.loads(tool_call.function.arguments)
            print(f"  🔧 调用工具: {name}({args})")
            result = execute_tool(name, args)
            print(f"  📋 工具返回: {result}")
            messages.append({
                "role": "tool",
                "tool_call_id": tool_call.id,
                "content": result
            })

    if not agent_answer:
        return "执行阶段未能生成回答"

    # ===== 阶段二：反思，对初版回答进行自我审查 =====
    print(f"\n{'─'*40}")
    print("🔍 阶段二：自我反思，审查初版回答")
    print(f"{'─'*40}")

    reflection_response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{
            "role": "user",
            "content": REFLECTION_PROMPT.format(
                user_input=user_input,
                agent_answer=agent_answer
            )
        }]
    )
    reflection = reflection_response.choices[0].message.content
    print(f"\n🔍 质检报告:\n{reflection}")

    # ===== 阶段三：如果质检不通过，修正回答 =====
    needs_revision = any(grade in reflection for grade in ["评分：C", "评分：D", "评分: C", "评分: D"])

    if needs_revision:
        print(f"\n{'─'*40}")
        print("🔄 阶段三：修正回答")
        print(f"{'─'*40}")

        revision_response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{
                "role": "user",
                "content": REVISION_PROMPT.format(
                    user_input=user_input,
                    agent_answer=agent_answer,
                    reflection=reflection
                )
            }]
        )
        final_answer = revision_response.choices[0].message.content
        print(f"\n✅ 修正后的最终回答:\n{final_answer}")
        return final_answer
    else:
        print(f"\n✅ 质检通过！初版回答质量合格，无需修正。")
        return agent_answer
```

注意看质检报告的第 5 条——它揪出了一个**人也很容易忽略的问题**：高铁票 800 元到底是单程还是往返？如果是单程，总费用直接翻一截。

这就是反思的价值。它不是在挑刺，而是在帮你兜底。如果初版回答有更严重的问题（比如漏掉了某个子任务），质检会打 C 或 D 分，直接触发修正流程自动补全。

## 九、接下来可以折腾啥？

跑通上面三个版本之后，你对智能体的核心套路已经心里有数了。下一步可以挑感兴趣的方向继续挖：

**加更多工具** — 搜索引擎、读写文件、发邮件、查数据库……写法跟前面一样：写函数 → 写 JSON 说明书 → 注册进去，就这么简单。

**加长期记忆** — 现在的记忆是"阅后即焚"，对话结束就没了。用 SQLite 或向量数据库（ChromaDB 之类）把历史存下来，下次还能接着聊。

**多智能体协作** — V3 其实已经有多角色的雏形了（执行者 + 质检员）。再往前走一步，可以让多个智能体分工：一个搜资料，一个写代码，一个做审核。

**Planning 模式** — 在 ReAct 前面再加一步"先列计划"。任务特别复杂的时候（10 步以上），先输出一份执行计划和依赖关系，再按计划干活，翻车率会低很多。

**接入 MCP 协议** — Anthropic 在 2025 年搞的一套标准化工具接入协议。简单理解就是：以前给智能体加工具得自己写 JSON 说明书，用了 MCP 之后各种工具可以"即插即用"，像给电脑插 USB 一样。

## 写在最后

回头看这三个版本，代码加在一起也没多少行，但每一次升级解决的都是真实的痛点：V1 解决"能不能干活"，V2 解决"出了问题怎么排查"，V3 解决"怎么在交付前少犯错"。

其实这跟我们带新人的过程挺像的——新人刚来先让他能把活干完（V1），然后要求他做事之前先说思路（V2），最后培养他自己检查一遍再提交的习惯（V3）。

动手跑跑看吧，把代码复制下来，先跑通，然后试着加一个你自己想要的工具——比如让它能帮你查快递、帮你翻译、帮你读 PDF。真正自己跑一遍，比看十篇文章都管用。

有什么问题欢迎评论区聊，我们下篇见。
