第 5 篇咱们搞了记忆系统——滑动窗口、摘要压缩、长期记忆。第 2 篇搞了 RAG——从文档库里检索相关信息。但这两件事一直是各干各的：记忆管对话历史，RAG 管外部知识，中间没有交集。

问题来了：用户跟你的智能体聊了 50 次，提到过"我对 Go 语言过敏，别推荐 Go"——下次用户问"推荐个后端语言"，RAG 从文档库里检索到一堆 Go 的文章，噼里啪啦推了一通。用户体验直接崩了。

如果记忆和检索能**融合**呢？检索的时候不仅看文档库，还看用户的历史偏好；回答的时候不仅基于检索结果，还基于对这个用户的"了解"。这就是**记忆增强检索（Memory-Augmented Retrieval）**。

说白了，就是让智能体**越用越聪明**——它不仅知道"世界上有什么"，还知道"你是谁、你要什么"。

## 一、记忆和检索为什么要融合？

### 1.1 两个系统各自的局限

| | 纯记忆（第 5 篇） | 纯 RAG（第 2 篇） |
|---|---|---|
| **擅长** | 记住用户偏好、对话上下文 | 从海量文档中找相关信息 |
| **不擅长** | 搜索外部知识 | 个性化、理解用户意图 |
| **核心问题** | "知道你是谁，但不知道世界上有什么" | "知道世界上有什么，但不知道你是谁" |

### 1.2 融合后的效果

```
用户（第 1 次对话）：我是做 Python 后端的，对性能很敏感，讨厌 Java 的冗长。

用户（第 30 次对话）：推荐个数据库方案

纯 RAG 的回答：根据文档，主流数据库有 MySQL、PostgreSQL、MongoDB……（泛泛而谈）

记忆增强检索的回答：根据你之前提到的偏好——Python 后端、重视性能——我推荐 PostgreSQL
配合 asyncpg 驱动，异步性能极佳。如果是高写入场景，ScyllaDB 也值得考虑（它比
Cassandra 性能更好，而且 Python 驱动更友好）。没推 Java 系的 Cassandra 和 HBase
因为你之前说过不喜欢 Java 生态。
```

差别一目了然：融合后的智能体**懂你**。

## 二、两种记忆：情景记忆 vs 语义记忆

人的长期记忆分两种，智能体也可以这么分：

**情景记忆（Episodic Memory）**：具体的经历。"上周二用户问了关于 FastAPI 部署的问题，我推荐了 Docker，用户说好用。"每一条都是一个"事件"，有时间、有上下文。

**语义记忆（Semantic Memory）**：从经历中提炼的知识。"这个用户偏好 Python 技术栈，重视性能，不喜欢 Java。"没有具体时间，是抽象出来的"认知"。

```python
import os
import json
import time
from openai import OpenAI
import chromadb

client = OpenAI(
    base_url="https://api.deepseek.com",
    api_key=os.getenv("DEEPSEEK_API_KEY")
)
MODEL_NAME = "deepseek-chat"


class EpisodicMemory:
    """情景记忆：存储具体的对话事件"""

    def __init__(self):
        chroma = chromadb.Client()
        self.collection = chroma.get_or_create_collection(
            "episodic_memory",
            metadata={"hnsw:space": "cosine"}
        )
        self._counter = 0

    def store(self, user_msg: str, assistant_msg: str, metadata: dict | None = None) -> None:
        """存储一轮对话"""
        text = f"用户：{user_msg}\n助手：{assistant_msg}"
        meta = {
            "timestamp": time.time(),
            "user_msg": user_msg[:200],
            "assistant_msg": assistant_msg[:200],
            **(metadata or {})
        }
        self._counter += 1
        self.collection.add(
            ids=[f"ep_{self._counter}"],
            documents=[text],
            metadatas=[meta]
        )

    def recall(self, query: str, top_k: int = 3) -> list[dict]:
        """根据语义相似性回忆相关的历史对话"""
        if self.collection.count() == 0:
            return []
        results = self.collection.query(
            query_texts=[query],
            n_results=min(top_k, self.collection.count())
        )
        memories = []
        if results["documents"]:
            for doc, meta in zip(results["documents"][0], results["metadatas"][0]):
                memories.append({"content": doc, "metadata": meta})
        return memories


class SemanticMemory:
    """语义记忆：存储从对话中提炼的用户认知"""

    def __init__(self):
        chroma = chromadb.Client()
        self.collection = chroma.get_or_create_collection(
            "semantic_memory",
            metadata={"hnsw:space": "cosine"}
        )
        self._counter = 0

    def extract_and_store(self, conversation: str) -> list[str]:
        """从对话中提取用户偏好/事实，存入语义记忆"""
        response = client.chat.completions.create(
            model=MODEL_NAME,
            messages=[{
                "role": "user",
                "content": (
                    "从以下对话中提取关于用户的关键事实和偏好。"
                    "每条事实独占一行，用 - 开头。"
                    "只提取明确表达的事实，不要推测。"
                    "如果没有值得提取的信息，返回"无"。\n\n"
                    f"对话内容：\n{conversation}"
                )
            }],
            temperature=0
        )

        text = response.choices[0].message.content
        if "无" in text and len(text) < 10:
            return []

        facts = [line.strip().lstrip("- ") for line in text.strip().split("\n") if line.strip().startswith("-")]

        for fact in facts:
            if len(fact) < 3:
                continue
            self._counter += 1
            existing = self.collection.query(query_texts=[fact], n_results=1)
            # 避免重复存储相似的事实
            if existing["distances"] and existing["distances"][0] and existing["distances"][0][0] < 0.3:
                continue
            self.collection.add(
                ids=[f"sem_{self._counter}"],
                documents=[fact],
                metadatas=[{"timestamp": time.time(), "source": "extraction"}]
            )

        return facts

    def recall(self, query: str, top_k: int = 5) -> list[str]:
        """检索与查询相关的用户认知"""
        if self.collection.count() == 0:
            return []
        results = self.collection.query(
            query_texts=[query],
            n_results=min(top_k, self.collection.count())
        )
        if results["documents"]:
            return results["documents"][0]
        return []


# 测试两种记忆
episodic = EpisodicMemory()
semantic = SemanticMemory()

# 模拟几轮对话
conversations = [
    ("我是做 Python 后端的，主要用 FastAPI", "了解！FastAPI 确实很适合高性能 API 开发。"),
    ("我对性能很敏感，之前用过 Django 觉得太重了", "理解，FastAPI 确实比 Django 轻量很多，异步性能也更好。"),
    ("数据库我一直用 PostgreSQL，挺满意的", "PostgreSQL 确实是 Python 后端的好搭档，尤其配合 asyncpg。"),
    ("千万别给我推荐 Java 的东西，我受不了那个语法", "明白，完全理解你的偏好。"),
]

for user_msg, assistant_msg in conversations:
    episodic.store(user_msg, assistant_msg)
    semantic.extract_and_store(f"用户：{user_msg}\n助手：{assistant_msg}")

# 检索测试
print("=== 情景记忆回忆 ===")
episodes = episodic.recall("推荐一个数据库")
for ep in episodes:
    print(f"  {ep['content'][:80]}...")

print("\n=== 语义记忆回忆 ===")
facts = semantic.recall("推荐后端技术栈")
for fact in facts:
    print(f"  - {fact}")
```

## 三、记忆增强的 RAG 检索

核心思路：**检索的时候，不仅从文档库找相关知识，还从记忆中找用户偏好，然后把两者融合后喂给 LLM**。

### 3.1 记忆感知的检索改写

用户问"推荐个数据库"，直接拿这句话去检索，结果是泛泛的。但如果先用记忆改写查询：

```python
class MemoryAwareRetriever:
    """记忆感知的检索器"""

    def __init__(self, knowledge_base: chromadb.Collection, semantic_memory: SemanticMemory):
        self.kb = knowledge_base
        self.memory = semantic_memory

    def rewrite_query(self, original_query: str) -> str:
        """用语义记忆改写检索查询"""
        user_facts = self.memory.recall(original_query, top_k=3)
        if not user_facts:
            return original_query

        facts_text = "\n".join(f"- {f}" for f in user_facts)
        response = client.chat.completions.create(
            model=MODEL_NAME,
            messages=[{
                "role": "user",
                "content": (
                    "根据用户的已知偏好，改写以下检索查询，使其更精准。\n"
                    "不要改变原始意图，只是加入用户偏好作为约束。\n"
                    "直接返回改写后的查询，不要解释。\n\n"
                    f"用户偏好：\n{facts_text}\n\n"
                    f"原始查询：{original_query}"
                )
            }],
            temperature=0
        )
        return response.choices[0].message.content.strip()

    def retrieve(self, query: str, top_k: int = 3) -> list[dict]:
        """记忆增强的检索"""
        # 1. 用记忆改写查询
        rewritten = self.rewrite_query(query)
        print(f"  原始查询：{query}")
        print(f"  改写后：{rewritten}")

        # 2. 用改写后的查询检索知识库
        results = self.kb.query(query_texts=[rewritten], n_results=top_k)
        docs = []
        if results["documents"]:
            for doc, dist in zip(results["documents"][0], results["distances"][0]):
                docs.append({"content": doc, "distance": dist})
        return docs


# 初始化知识库
chroma_client = chromadb.Client()
knowledge = chroma_client.get_or_create_collection("tech_knowledge")

tech_docs = [
    "PostgreSQL 是强大的开源关系型数据库，支持 JSON、全文搜索，asyncpg 驱动性能极高。",
    "MySQL 是最流行的开源数据库，广泛用于 PHP 和 Java 项目。",
    "MongoDB 是 NoSQL 文档数据库，适合灵活的 schema 和快速迭代。",
    "Redis 是内存键值存储，常用于缓存和会话管理，Python 有 aioredis 异步驱动。",
    "Cassandra 是 Java 生态的分布式数据库，适合超大规模写入。",
    "ScyllaDB 是 Cassandra 的 C++ 重写版，性能提升 10 倍，Python 驱动友好。",
    "SQLite 是嵌入式数据库，适合本地应用和小型项目，Python 标准库内置支持。",
    "FastAPI + PostgreSQL + asyncpg 是 Python 高性能后端的经典组合。",
]

existing_ids = knowledge.get()["ids"]
new_docs = [(f"tech_{i}", doc) for i, doc in enumerate(tech_docs) if f"tech_{i}" not in existing_ids]
if new_docs:
    knowledge.add(
        ids=[x[0] for x in new_docs],
        documents=[x[1] for x in new_docs]
    )

# 测试记忆感知检索
retriever = MemoryAwareRetriever(knowledge, semantic)

print("\n=== 记忆增强检索 ===")
docs = retriever.retrieve("推荐个数据库方案")
for doc in docs:
    print(f"  [{doc['distance']:.3f}] {doc['content'][:80]}...")
```

对比：
- 纯 RAG 用"推荐个数据库方案"检索，可能返回 MySQL、Cassandra 等泛结果
- 记忆增强检索先改写为"Python 后端、高性能、非 Java 生态的数据库推荐"，检索结果更精准

## 四、对话感知的 RAG

除了用语义记忆改写查询，还有一种更直接的方式：**用对话上下文来增强检索**。

### 4.1 上下文感知检索

用户连续对话时，后面的问题往往依赖前面的上下文。比如用户先问"FastAPI 怎么样？"，然后问"它的性能数据呢？"——这里的"它"指的是 FastAPI，但如果直接拿"它的性能数据呢？"去检索，结果可能风马牛不相及。

```python
class ConversationAwareRetriever:
    """对话感知的检索器：利用对话上下文改写查询"""

    def __init__(self, knowledge_base: chromadb.Collection):
        self.kb = knowledge_base

    def contextualize_query(self, current_query: str, recent_history: list[dict]) -> str:
        """结合对话历史，把指代不清的查询改写成自包含的查询"""
        if not recent_history:
            return current_query

        history_text = "\n".join(
            f"{m['role']}：{m['content']}"
            for m in recent_history[-6:]
            if m["role"] != "system"
        )

        response = client.chat.completions.create(
            model=MODEL_NAME,
            messages=[{
                "role": "user",
                "content": (
                    "根据对话历史，把用户最新的问题改写成一个独立的、完整的查询。\n"
                    "解决代词指代（如'它'、'这个'）和省略的主语。\n"
                    "直接返回改写后的查询，不要解释。\n\n"
                    f"对话历史：\n{history_text}\n\n"
                    f"最新问题：{current_query}"
                )
            }],
            temperature=0
        )
        return response.choices[0].message.content.strip()

    def retrieve(self, query: str, history: list[dict], top_k: int = 3) -> list[dict]:
        """对话感知的检索"""
        contextualized = self.contextualize_query(query, history)
        print(f"  原始：{query}")
        print(f"  改写：{contextualized}")

        results = self.kb.query(query_texts=[contextualized], n_results=top_k)
        docs = []
        if results["documents"]:
            for doc, dist in zip(results["documents"][0], results["distances"][0]):
                docs.append({"content": doc, "distance": dist})
        return docs


# 测试
conv_retriever = ConversationAwareRetriever(knowledge)

history = [
    {"role": "user", "content": "FastAPI 怎么样？"},
    {"role": "assistant", "content": "FastAPI 是一个高性能的 Python Web 框架，基于 Starlette，支持异步。"},
    {"role": "user", "content": "它一般配什么数据库用？"},
    {"role": "assistant", "content": "FastAPI 常配合 PostgreSQL 使用，用 asyncpg 做异步连接。"},
]

print("\n=== 对话感知检索 ===")
docs = conv_retriever.retrieve("有没有性能测试数据？", history)
for doc in docs:
    print(f"  [{doc['distance']:.3f}] {doc['content'][:80]}...")
```

## 五、完整方案：记忆增强的智能体

把情景记忆、语义记忆、记忆感知检索、对话感知检索全部融合，做一个完整的"越用越聪明"的智能体：

```python
class MemoryAugmentedAgent:
    """记忆增强智能体：融合记忆与检索"""

    def __init__(self):
        self.episodic = EpisodicMemory()
        self.semantic = SemanticMemory()

        # 知识库
        chroma_client = chromadb.Client()
        self.knowledge = chroma_client.get_or_create_collection("agent_knowledge")
        self._init_knowledge()

        # 当前会话历史
        self.session_history: list[dict] = []

    def _init_knowledge(self) -> None:
        """初始化知识库"""
        docs = [
            "PostgreSQL 是强大的关系型数据库，asyncpg 驱动性能极高，适合 Python 异步后端。",
            "FastAPI + PostgreSQL + asyncpg 是 Python 高性能后端的经典黄金组合。",
            "Django 是全功能 Web 框架，自带 ORM 和 Admin，适合快速开发复杂应用。",
            "Redis 是内存缓存，Python 可用 aioredis 做异步缓存层。",
            "Docker 是容器化部署的标准工具，配合 Docker Compose 管理多服务。",
            "Kubernetes 用于大规模容器编排，适合微服务架构。",
            "pytest 是 Python 最流行的测试框架，支持异步测试和参数化。",
            "MySQL 是最流行的开源数据库，Java 和 PHP 生态使用最广。",
        ]
        ids = [f"k_{i}" for i in range(len(docs))]
        existing = self.knowledge.get()["ids"]
        new_items = [(id_, doc) for id_, doc in zip(ids, docs) if id_ not in existing]
        if new_items:
            self.knowledge.add(
                ids=[x[0] for x in new_items],
                documents=[x[1] for x in new_items]
            )

    def _retrieve_with_memory(self, query: str) -> list[str]:
        """记忆增强的检索"""
        # 1. 对话感知改写（解决指代）
        if self.session_history:
            recent = self.session_history[-6:]
            history_text = "\n".join(
                f"{m['role']}：{m['content']}" for m in recent if m["role"] != "system"
            )
            rewrite_response = client.chat.completions.create(
                model=MODEL_NAME,
                messages=[{
                    "role": "user",
                    "content": (
                        "把以下问题改写成独立的完整查询，解决指代和省略。"
                        "直接返回改写结果。\n\n"
                        f"对话历史：\n{history_text}\n\n问题：{query}"
                    )
                }],
                temperature=0
            )
            contextualized_query = rewrite_response.choices[0].message.content.strip()
        else:
            contextualized_query = query

        # 2. 语义记忆增强（加入用户偏好）
        user_facts = self.semantic.recall(contextualized_query, top_k=3)
        if user_facts:
            facts_text = "；".join(user_facts)
            enhanced_query = f"{contextualized_query}（用户偏好：{facts_text}）"
        else:
            enhanced_query = contextualized_query

        # 3. 检索知识库
        results = self.knowledge.query(
            query_texts=[enhanced_query],
            n_results=3
        )

        # 4. 检索情景记忆（相关的历史对话）
        episodes = self.episodic.recall(query, top_k=2)

        retrieved = []
        if results["documents"]:
            for doc in results["documents"][0]:
                retrieved.append(f"[知识库] {doc}")
        for ep in episodes:
            retrieved.append(f"[历史对话] {ep['content']}")

        return retrieved

    def chat(self, user_input: str) -> str:
        """对话入口"""
        # 检索
        context_docs = self._retrieve_with_memory(user_input)

        # 构建用户偏好摘要
        user_facts = self.semantic.recall(user_input, top_k=5)
        user_profile = ""
        if user_facts:
            user_profile = "已知用户信息：\n" + "\n".join(f"- {f}" for f in user_facts) + "\n\n"

        # 构建上下文
        context_text = "\n\n".join(context_docs) if context_docs else "暂无参考资料"

        messages = [
            {
                "role": "system",
                "content": (
                    "你是一个技术顾问。基于参考资料和用户的已知偏好回答问题。\n"
                    "如果知道用户的偏好，回答时要考虑这些偏好。\n"
                    "如果参考资料不足，明确告知。"
                )
            }
        ]

        # 加入最近的对话历史（保持连贯）
        messages.extend(self.session_history[-6:])

        # 当前问题 + 上下文
        user_message = f"{user_profile}参考资料：\n{context_text}\n\n问题：{user_input}"
        messages.append({"role": "user", "content": user_message})

        # 调用 LLM
        response = client.chat.completions.create(
            model=MODEL_NAME,
            messages=messages,
            temperature=0.7
        )
        answer = response.choices[0].message.content

        # 更新记忆
        self.session_history.append({"role": "user", "content": user_input})
        self.session_history.append({"role": "assistant", "content": answer})

        self.episodic.store(user_input, answer)
        self.semantic.extract_and_store(f"用户：{user_input}\n助手：{answer}")

        return answer


# —— 运行 ——

# 第一轮对话：建立用户画像
```

## 六、记忆的生命周期管理

记忆不能只加不减，时间久了噪音太多。需要管理记忆的生命周期。

### 6.1 记忆衰减

越久远的记忆权重越低，模拟人的遗忘曲线：

```python
import math


def decay_score(base_score: float, hours_ago: float, half_life: float = 168.0) -> float:
    """
    记忆衰减函数
    base_score: 原始相关性分数
    hours_ago: 距今多少小时
    half_life: 半衰期（默认 7 天 = 168 小时）
    """
    decay = math.exp(-0.693 * hours_ago / half_life)
    return base_score * decay


# 同一条记忆在不同时间点的权重
base = 0.9
for days in [0, 1, 7, 30, 90]:
    score = decay_score(base, hours_ago=days * 24)
    print(f"  {days:3d} 天前的记忆：{base:.2f} → {score:.3f}")
```

### 6.2 记忆合并与去重

当语义记忆中有相似或矛盾的条目时，需要合并：

```python
def merge_memories(memories: list[str]) -> list[str]:
    """合并相似或矛盾的记忆"""
    if len(memories) <= 1:
        return memories

    memories_text = "\n".join(f"{i+1}. {m}" for i, m in enumerate(memories))

                "1. 合并相似的条目\n"
                "2. 如果有矛盾，保留最新的（编号大的更新）\n"

                f"记忆条目：\n{memories_text}"

        temperature=0

    text = response.choices[0].message.content
    return [line.strip().lstrip("- ") for line in text.strip().split("\n") if line.strip().startswith("-")]
```

### 6.3 记忆容量管理

给记忆设上限，超出后删除最不重要的：

```python
def prune_memories(
    collection: chromadb.Collection,
    max_count: int = 100,
    keep_recent: int = 20
) -> int:
    """修剪记忆：保留最近的 + 最重要的，删除其余的"""
    current = collection.count()
    if current <= max_count:
        return 0

    all_data = collection.get(include=["metadatas"])
    ids = all_data["ids"]
    metas = all_data["metadatas"]

    # 按时间排序
    timed = sorted(
        zip(ids, metas),
        key=lambda x: x[1].get("timestamp", 0),
        reverse=True
    )

    # 最近的一定保留
    keep_ids = set(id_ for id_, _ in timed[:keep_recent])

    # 其余的按衰减分数排序，保留到 max_count
    remaining = [(id_, meta) for id_, meta in timed[keep_recent:]]
    now = time.time()
    scored = []
    for id_, meta in remaining:
        hours_ago = (now - meta.get("timestamp", now)) / 3600
        score = decay_score(1.0, hours_ago)
        scored.append((score, id_))

    scored.sort(reverse=True)
    spots_left = max_count - len(keep_ids)
    keep_ids.update(id_ for _, id_ in scored[:spots_left])

    # 删除不保留的
    delete_ids = [id_ for id_ in ids if id_ not in keep_ids]
    if delete_ids:
        collection.delete(ids=delete_ids)

    return len(delete_ids)
```

## 七、常见坑

### 坑 1：什么都往记忆里存

"用户说了一句'好的'也存？"——不是所有对话都有记忆价值。只存包含实质信息的对话（偏好、决策、事实），闲聊和确认类的别存。

### 坑 2：不做记忆去重

用户说了 5 次"我喜欢 Python"，语义记忆里就存 5 条。检索的时候 5 条全返回，浪费 token。一定要做去重和合并。

### 坑 3：检索改写过度

记忆增强改写查询时，加太多偏好约束，反而限制了检索范围。改写要**适度**——加入关键偏好就行，别把用户画像全塞进去。

### 坑 4：忽略矛盾处理

用户第 1 次说"我喜欢 Django"，第 30 次说"我觉得 Django 太重了"。如果不处理矛盾，两条记忆同时存在，回答就会自相矛盾。新记忆应该能覆盖旧记忆。

### 坑 5：没有遗忘机制

记忆只增不减，越积越多。3 个月前的偏好可能早就变了，但还在记忆里影响回答。一定要加衰减和修剪。

## 八、总结

| 技术 | 解决什么问题 | 核心思路 |
|------|-------------|----------|
| **情景记忆** | 记住具体的对话事件 | 向量化存储每轮对话，语义检索回忆 |
| **语义记忆** | 提炼用户画像 | LLM 从对话中抽取事实，持久化存储 |
| **记忆感知检索** | 检索结果不个性化 | 用用户偏好改写检索查询 |
| **对话感知检索** | 指代不清导致检索偏差 | 用对话上下文改写查询 |
| **记忆衰减** | 旧记忆噪音 | 时间越久权重越低 |
| **记忆合并** | 重复和矛盾 | LLM 做去重和冲突解决 |

记忆和检索的融合，本质上是让智能体从"搜索引擎模式"升级为"个人助手模式"——它不仅知道世界上有什么，还知道你是谁、你要什么。这才是真正"越用越聪明"的 AI。
