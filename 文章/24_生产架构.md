## 一、Demo 和生产差在哪？

你的 Demo 大概长这样：`main.py` 一个文件，直接调 OpenAI API，对话历史存在一个 Python 列表里，工具是几个模拟函数，安全机制没有，监控就是 `print()`。能跑，效果也不错。

但你把它给 100 个人用试试？

第一个问题：**模型 API 挂了**。Demo 阶段你手动重启就好，生产环境里用户正在等着回复，你的服务直接抛了个 500。你需要重试机制和备用模型降级。

第二个问题：**服务重启丢记忆**。对话历史存在内存里，部署了新版本一重启，所有用户的对话上下文全没了。你需要会话持久化。

第三个问题：**十个用户同时请求**。你的单线程 Demo 一次只能处理一个，其他人排队等着。你需要异步处理和并发控制。

第四个问题：**月底账单**。没有 token 预算控制，某个用户拿你的 Agent 当聊天室，一天聊了 500 轮，月底多出来 $200。

这些问题不是写几行代码能解决的——它们需要一张架构图来系统性地覆盖。

## 三、关键组件详解

### 3.1 网关层：第一道门

```python
import time
from collections import defaultdict


class APIGateway:
    """API 网关：认证 + 限流"""

    def __init__(self, rate_limit: int = 60):
        self.rate_limit = rate_limit  # 每分钟最多请求数
        self.request_counts: dict[str, list[float]] = defaultdict(list)

    def authenticate(self, api_key: str) -> dict | None:
        """认证：验证 API Key"""
        valid_keys = {
            "sk-test-001": {"user_id": "user_001", "tier": "free", "daily_limit": 100},
            "sk-test-002": {"user_id": "user_002", "tier": "pro", "daily_limit": 10000},
        }
        return valid_keys.get(api_key)

    def rate_limit_check(self, user_id: str) -> bool:
        """限流：每分钟请求数检查"""
        now = time.time()
        # 清理 1 分钟前的记录
        self.request_counts[user_id] = [
            t for t in self.request_counts[user_id] if now - t < 60
        ]
        if len(self.request_counts[user_id]) >= self.rate_limit:
            return False
        self.request_counts[user_id].append(now)
        return True

    def process(self, api_key: str, request: dict) -> dict:
        """处理请求"""
        # 认证
        user = self.authenticate(api_key)
        if not user:
            return {"error": "无效的 API Key", "code": 401}

        # 限流
        if not self.rate_limit_check(user["user_id"]):
            return {"error": "请求过于频繁，请稍后再试", "code": 429}

        return {"user": user, "request": request, "code": 200}
```

### 3.2 模型调用层：比你想的复杂

生产环境的模型调用不是 `client.chat.completions.create()` 一行了事。要处理：

```python
import random


class LLMGateway:
    """模型网关：重试 + 降级 + 路由"""

    def __init__(self):
        self.models = {
            "primary": {"name": "deepseek-chat", "base_url": "https://api.deepseek.com"},
            "fallback": {"name": "deepseek-chat", "base_url": "https://api.deepseek.com"},
        }
        self.retry_config = {"max_retries": 3, "base_delay": 1.0}

    def call(self, messages: list[dict], model_preference: str = "primary") -> dict:
        """带重试和降级的模型调用"""
        models_to_try = [model_preference, "fallback"] if model_preference != "fallback" else ["fallback"]

        for model_key in models_to_try:
            model = self.models.get(model_key)
            if not model:
                continue

            for attempt in range(self.retry_config["max_retries"]):
                try:
                    # 模拟调用（实际项目中替换为真实 API 调用）
                    result = self._do_call(model, messages)
                    return {
                        "content": result,
                        "model_used": model_key,
                        "attempts": attempt + 1
                    }
                except Exception as e:
                    delay = self.retry_config["base_delay"] * (2 ** attempt)
                    # 指数退避：1s, 2s, 4s
                    if attempt < self.retry_config["max_retries"] - 1:
                        time.sleep(delay)
                    else:
                        continue  # 试下一个模型

        return {"content": "所有模型均不可用，请稍后再试", "model_used": "none", "attempts": 0}

    def _do_call(self, model: dict, messages: list[dict]) -> str:
        """模拟 LLM 调用（实际替换为 OpenAI SDK）"""
        # 模拟 5% 的失败率
        if random.random() < 0.05:
            raise ConnectionError("模型服务暂时不可用")
        return f"[{model['name']}] 模拟回复"
```

### 3.3 会话管理：不只是 messages 列表

```python
import json
import uuid
from pathlib import Path


class SessionManager:
    """会话管理器：持久化 + 过期 + 隔离"""

    def __init__(self, storage_dir: str = "/tmp/agent_sessions"):
        self.storage_dir = Path(storage_dir)
        self.storage_dir.mkdir(parents=True, exist_ok=True)

    def create_session(self, user_id: str) -> str:
        """创建新会话"""
        session_id = str(uuid.uuid4())
        session = {
            "session_id": session_id,
            "user_id": user_id,
            "created_at": time.time(),
            "last_active": time.time(),
            "messages": [],
            "metadata": {}
        }
        self._save(session_id, session)
        return session_id

    def get_session(self, session_id: str) -> dict | None:
        """获取会话"""
        return self._load(session_id)

    def add_message(self, session_id: str, role: str, content: str) -> None:
        """添加消息"""
        session = self._load(session_id)
        if not session:
            return
        session["messages"].append({"role": role, "content": content, "timestamp": time.time()})
        session["last_active"] = time.time()
        self._save(session_id, session)

    def cleanup_expired(self, max_age_hours: int = 24) -> int:
        """清理过期会话"""
        cleaned = 0
        now = time.time()
        for f in self.storage_dir.glob("*.json"):
            session = json.loads(f.read_text())
            if now - session["last_active"] > max_age_hours * 3600:
                f.unlink()
                cleaned += 1
        return cleaned

    def _save(self, session_id: str, data: dict) -> None:
        path = self.storage_dir / f"{session_id}.json"
        path.write_text(json.dumps(data, ensure_ascii=False))

    def _load(self, session_id: str) -> dict | None:
        path = self.storage_dir / f"{session_id}.json"
        if path.exists():
            return json.loads(path.read_text())
        return None
```

### 3.4 编排层：整个系统的大脑

```python
class Orchestrator:
    """编排器：串联所有组件"""

    def __init__(
        self,
        llm_gateway: LLMGateway,
        session_mgr: SessionManager,
        gateway: APIGateway
    ):
        self.llm = llm_gateway
        self.sessions = session_mgr
        self.gateway = gateway

    def handle_request(self, api_key: str, session_id: str | None, user_input: str) -> dict:
        """处理一次完整的请求"""

        # 1. 网关：认证 + 限流
        gate_result = self.gateway.process(api_key, {"input": user_input})
        if gate_result["code"] != 200:
            return gate_result

        user = gate_result["user"]

        # 2. 会话管理
        if not session_id:
            session_id = self.sessions.create_session(user["user_id"])
        session = self.sessions.get_session(session_id)
        if not session:
            session_id = self.sessions.create_session(user["user_id"])
            session = self.sessions.get_session(session_id)

        # 3. 构建消息
        messages = [
            {"role": "system", "content": "你是一个助手。"},
            *session["messages"][-10:],  # 最近 5 轮
            {"role": "user", "content": user_input}
        ]

        # 4. 调用模型（带重试和降级）
        llm_result = self.llm.call(messages)

        # 5. 保存消息
        self.sessions.add_message(session_id, "user", user_input)
        self.sessions.add_message(session_id, "assistant", llm_result["content"])

        return {
            "code": 200,
            "session_id": session_id,
            "response": llm_result["content"],
            "model_used": llm_result["model_used"]
        }


# 组装
orchestrator = Orchestrator(
    llm_gateway=LLMGateway(),
    session_mgr=SessionManager(),
    gateway=APIGateway()
)

# 模拟请求
result = orchestrator.handle_request(
    api_key="sk-test-002",
    session_id=None,
    user_input="Python 的 asyncio 怎么用？"
)
print(f"会话：{result.get('session_id', 'N/A')[:8]}...")
print(f"模型：{result.get('model_used', 'N/A')}")
print(f"回复：{result.get('response', result.get('error', 'N/A'))}")
```

## 四、扩展性设计

### 4.1 水平扩展的关键：无状态

编排层要设计成**无状态的**——状态都存在外部（数据库、Redis）。这样才能水平扩展：起 10 个实例，任何一个都能处理任何请求。

```
❌ 有状态：session 存在进程内存里
   → 用户第一次请求打到实例 A，第二次打到实例 B，B 找不到 session

✅ 无状态：session 存在 Redis/数据库里
   → 任何实例都能读到 session，随便扩缩
```

### 4.2 异步队列：削峰填谷

用户量大时，不能每个请求都同步等模型返回（模型调用可能要 5-30 秒）。用消息队列解耦：

```
同步模式（小规模）：
  用户 → API → 调模型（等 10 秒）→ 返回结果

异步模式（大规模）：
  用户 → API → 投入队列 → 立即返回任务 ID
  后台 Worker → 从队列取任务 → 调模型 → 写结果
  用户 → 轮询/WebSocket 获取结果
```

### 4.3 多租户隔离

如果你的智能体服务多个客户，每个客户的数据要隔离：

class TenantIsolation:
    """多租户隔离"""

## 五、上线前的检查清单

```
□ 基础设施
  □ 模型调用有重试和降级机制
  □ 会话数据持久化（不在进程内存里）
  □ 有健康检查接口（/health）

□ 安全
  □ API 需要认证（API Key / OAuth）
  □ 输入/输出有守门人检查
  □ 敏感数据不落日志

□ 可靠性
  □ 单个请求有超时限制（如 60 秒）
  □ 工具调用有超时和重试
  □ 模型不可用时有降级方案

□ 可观测性
  □ 每次请求有唯一 trace_id
  □ 关键环节有结构化日志
  □ 有基础的监控告警

□ 成本控制
  □ 有 token 用量统计
  □ 有每用户/每天的用量上限
  □ 简单任务有模型路由（小模型优先）

□ 用户体验
  □ 流式输出（SSE / WebSocket）
  □ 会话超时提醒
  □ 错误信息对用户友好（不暴露内部细节）
```

## 六、你真的需要这些吗？

看完上面的架构，可能会觉得"这也太重了"。确实重。但重点不在于你是否一步到位全部实现，而在于你**知道这些层的存在**。

我见过一种典型的翻车路径：团队直接把 Demo 代码部署到生产，前面不加网关（所以没有限流），中间不加 LLM Gateway（所以模型挂了整个服务就挂了），会话直接存在内存里（所以重启就丢了）。第一个月用户少，风平浪静。第二个月用户量上来了，某天 DeepSeek 的 API 抖了 10 分钟，整个系统返回 500，积压的重试请求把服务打爆了。

所以我的建议是分阶段加固：

**第一天就该有的**：API Key 认证 + 基础限流 + LLM 调用重试。三个东西，加起来 100 行代码，能挡住 80% 的生产事故。

**用户量上来后加**：会话持久化 + 模型降级 + 异步队列。

**业务稳定后加**：多租户隔离 + 完整的可观测性 + 自动化部署。

别一上来就画满整张架构图——但也别连第一层防线都没有就上线。
