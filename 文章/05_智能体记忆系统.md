你跟 ChatGPT 聊了一下午，聊到喜欢的餐厅、下周的出差计划、最近在学的技术栈。第二天打开新对话，它不认识你了。你问"我下周去哪来着？"——它一脸懵。

你的智能体也一样。上一篇我们搞了 Agentic RAG，让 AI 自己决定翻哪本书、查几次。但对话一结束，一切归零。下次用户再来，它不知道对方是谁、上次聊了什么、喜欢什么。

说白了，AI 有金鱼记忆。

这篇文章要解决的就是这个问题：怎么让 AI 记住"你是谁、上次聊了什么、你喜欢什么"？我们从最简单的对话历史管理，一步步搭到带长期记忆的"懂你"助手。智能体那篇提到的 messages 列表就是最原始的记忆，这篇来升级它。

依赖没变，`pip install openai chromadb` 就够了，代码直接能跑。

## 一、智能体的记忆到底是什么？

先用人来打个比方。你脑子里其实有好几种"记性"：

- **工作记忆**：你此刻正在想的事，比如心算 37×48 的时候脑子里暂存的那几个数字
- **短期记忆**：今天发生的事，中午吃了啥、下午开了啥会
- **长期记忆**：你是谁、经历过什么、喜欢什么、讨厌什么

智能体的记忆也能这么分：

| 人的记忆 | 智能体的对应 | 存哪 |
|----------|--------------|------|
| 工作记忆 | 当前这一轮对话的 messages 列表 | 内存，每次请求都传 |
| 短期记忆 | 一次会话内的对话历史 | 内存，会话结束就没了 |
| 长期记忆 | 跨会话的用户偏好、历史摘要 | 持久化存储（数据库、向量库） |

那为啥光靠 messages 列表不够？三个现实问题：

**Token 限制** — GPT-4o 的上下文窗口再大也就 128K，对话一长，前面的内容要么被截断，要么根本塞不进去。

**成本** — 每次请求都把整段历史塞进去，token 烧得飞快。长对话一天聊下来，API 费用能翻好几倍。

**噪音** — 几十轮对话全塞进去，大模型容易在无关信息里打转，反而影响当前问题的回答质量。

所以咱们得给智能体设计一套"记忆管理"——该记的记，该忘的忘，该压缩的压缩。后面几节会从最糙的滑动窗口开始，一路做到带向量库的长期记忆，每步都有完整代码。

## 二、最简方案：滑动窗口（保留最近 N 轮）

思路很简单：只保留最近 N 轮对话，超出的直接丢掉。就像你只能记住最近几件事，再往前就模糊了。

实现也简单，几行代码搞定：

```python
import os
import json
import time
from openai import OpenAI

os.environ["OPENAI_API_KEY"] = "你的API Key"
client = OpenAI()

def sliding_window_memory(messages, max_turns=10):
    """滑动窗口：只保留最近 N 轮对话"""
    system_msg = [m for m in messages if m["role"] == "system"]
    conversation = [m for m in messages if m["role"] != "system"]
    # 每轮 = 一组 user + assistant，保留最近 max_turns 轮
    if len(conversation) > max_turns * 2:
        conversation = conversation[-(max_turns * 2):]
    return system_msg + conversation

# 模拟：用户聊了 15 轮，窗口设为 10
messages = [{"role": "system", "content": "你是一个助手"}]
for i in range(15):
    messages.append({"role": "user", "content": f"第{i+1}轮：我喜欢川菜" if i == 0 else f"第{i+1}轮：继续聊"})
    messages.append({"role": "assistant", "content": f"第{i+1}轮回复"})

trimmed = sliding_window_memory(messages, max_turns=10)
user_msgs = [m["content"] for m in trimmed if m["role"] == "user"]
print(f"原始轮数: 15，保留轮数: {len(user_msgs)}")
print(f"保留的对话: {user_msgs[0][:20]}... 到 ...{user_msgs[-1][:20]}...")
```

**运行效果：**

```
原始轮数: 15，保留轮数: 10
保留的对话: 第6轮：继续聊... 到 ...第15轮：继续聊...
```

第 1 轮用户说的"我喜欢川菜"被窗口滑出去了。到第 15 轮用户问"推荐个餐厅"，AI 已经不知道你喜欢川菜了。

优点：实现简单，token 可控。缺点：对话一长，前面说过什么全忘了。适合简单对话，不适合需要"记住很久以前说过啥"的场景。很多开源框架的默认配置就是这么干的，先跑起来再说。

## 三、进阶方案：摘要压缩

滑动窗口的问题是"一刀切"——前面的内容直接丢，有用的信息也没了。换个思路：用大模型把早期对话**压缩成一段摘要**，只保留关键信息。摘要 + 最近 N 轮 = 完整上下文。

20 轮对话可能几千 token，压缩成 200 字摘要，token 能省 80% 以上。

```python
SUMMARIZE_PROMPT = """请将以下对话历史压缩成一段简洁的摘要，保留：
1. 用户提到的关键信息（姓名、偏好、需求）
2. 已经做出的重要决定
3. 待办事项或未解决的问题
丢弃闲聊和重复内容。

对话历史：
{conversation}

请输出摘要（控制在 200 字以内）："""

class SummaryMemory:
    def __init__(self, max_recent=6, summarize_threshold=12):
        self.messages = []
        self.summary = ""
        self.max_recent = max_recent
        self.summarize_threshold = summarize_threshold

    def add(self, role, content):
        self.messages.append({"role": role, "content": content})
        if len(self.messages) > self.summarize_threshold:
            self._compress()

    def _compress(self):
        old_msgs = self.messages[:-self.max_recent]
        old_text = "\n".join([f"{m['role']}: {m['content']}" for m in old_msgs])
        response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": SUMMARIZE_PROMPT.format(conversation=old_text)}],
            temperature=0
        )
        self.summary = response.choices[0].message.content
        self.messages = self.messages[-self.max_recent:]

    def get_messages(self, system_prompt):
        context = f"{system_prompt}\n\n【之前的对话摘要】{self.summary}" if self.summary else system_prompt
        return [{"role": "system", "content": context}] + self.messages

# 模拟运行
mem = SummaryMemory(max_recent=4, summarize_threshold=8)
base = "你是一个助手。"
for i in range(10):
    if i == 0:
        mem.add("user", "我叫小明，喜欢川菜，下周要去上海出差")
        mem.add("assistant", "好的，记住了。需要帮你查上海的美食吗？")
    else:
        mem.add("user", f"随便聊聊{i}")
        mem.add("assistant", f"好的{i}")

msgs = mem.get_messages(base)
print("System 内容前 150 字:", msgs[0]["content"][:150])
print("最近对话轮数:", (len(msgs) - 1) // 2)
```

**运行效果：**

```
System 内容前 150 字: 你是一个助手。

【之前的对话摘要】用户名为小明，偏好川菜，计划下周出差上海。助手已确认并询问是否需要推荐上海美食。后续为一般性闲聊。

最近对话轮数: 4
```

第 20 轮用户问"我下周去哪来着？"，AI 能从摘要里找到"出差上海"，不会忘。摘要的质量取决于你给的 prompt，上面那个模板已经能覆盖大部分场景。如果你有特殊需求（比如必须保留数字、日期），在 prompt 里加一条就行。

## 四、长期记忆：跨会话的向量化存储

摘要压缩只在**单次会话**内有效。关掉窗口，下次打开，还是从零开始。

要跨会话记住用户，就得把关键信息**持久化**。存哪？向量数据库最合适——用户偏好、重要决定、历史问答摘要，都能存进去。下次对话开始，用用户的问题去向量库搜相关的历史记忆，把"关于这位用户的记忆"注入到 system prompt 里。

```python
import chromadb

chroma_client = chromadb.PersistentClient(path="./memory_db")

def get_embedding(text):
    response = client.embeddings.create(model="text-embedding-3-small", input=text)
    return response.data[0].embedding

class LongTermMemory:
    def __init__(self, user_id):
        self.collection = chroma_client.get_or_create_collection(
            name=f"user_{user_id}",
            metadata={"hnsw:space": "cosine"}
        )

    def save_memory(self, content, memory_type="preference"):
        """保存一条记忆"""
        self.collection.add(
            ids=[f"mem_{int(time.time()*1000)}"],
            documents=[content],
            metadatas=[{"type": memory_type, "timestamp": str(time.time())}],
            embeddings=[get_embedding(content)]
        )

    def recall(self, query, top_k=3):
        """根据当前对话回忆相关记忆"""
        results = self.collection.query(
            query_embeddings=[get_embedding(query)],
            n_results=top_k,
            include=["documents"]
        )
        return results["documents"][0] if results["documents"] else []

    def build_context(self, query, system_prompt):
        """把回忆到的记忆注入 system prompt"""
        memories = self.recall(query)
        if memories:
            memory_text = "\n".join([f"- {m}" for m in memories])
            return f"{system_prompt}\n\n【关于这位用户的记忆】\n{memory_text}"
        return system_prompt

# 模拟：第一天用户说了这些
ltm = LongTermMemory(user_id="xiaoming")
ltm.save_memory("用户名小明")
ltm.save_memory("职业前端开发")
ltm.save_memory("正在学习 Rust")

# 三天后新对话
query = "有什么适合我的技术文章推荐吗？"
ctx = ltm.build_context(query, "你是一个技术助手。")
print(ctx)
```

**运行效果：**

```
你是一个技术助手。

【关于这位用户的记忆】
- 正在学习 Rust
- 职业前端开发
- 用户名小明
```

AI 拿到这些记忆，就能推荐 Rust 入门 + 前端转全栈的文章，跨会话记住了。这里有个细节：`recall` 用的是语义检索，用户问"推荐学习资料"，会召回"正在学习 Rust""职业前端开发"这类相关记忆，而不是随机抓几条。这就是向量库的价值——按"意思"找，不按关键词匹配。

## 五、工作记忆：Scratchpad 模式

有些信息不是用户说的，是智能体自己推理出来的中间结论。比如做一道数学题，它先算出中间步骤，再得出最终答案。这些中间步骤如果不在 messages 里，下一轮就可能忘。

解决办法：给智能体一块**草稿纸**（Scratchpad）。在 system prompt 里加一块可读写的区域，每轮回答后，让它在回答末尾用 `[SCRATCHPAD]` 标记，写下需要记住的内容。咱们解析出来，下一轮塞回 prompt。

```python
SCRATCHPAD_PROMPT = """你有一块草稿纸，用来记录重要的中间信息。
每次回答后，如需记住某些内容，在回答末尾用 [SCRATCHPAD] 标记，后面写上要记住的内容。
给用户看的回答不要包含 [SCRATCHPAD] 及之后的内容。

【当前草稿纸】
{scratchpad}
"""

class WorkingMemory:
    def __init__(self):
        self.scratchpad = "（空）"

    def update_from_response(self, response_text):
        """从 AI 回复中提取草稿纸内容，返回给用户看的干净回复"""
        if "[SCRATCHPAD]" in response_text:
            parts = response_text.split("[SCRATCHPAD]")
            self.scratchpad = parts[1].strip()
            return parts[0].strip()
        return response_text

# 模拟
wm = WorkingMemory()
fake_response = """这道题需要先算 3+5=8，再算 8*2=16。
所以答案是 16。
[SCRATCHPAD]
已算出 3+5=8，8*2=16，最终答案 16
"""
clean = wm.update_from_response(fake_response)
print("给用户看的:", clean)
print("草稿纸:", wm.scratchpad)
```

**运行效果：**

```
给用户看的: 这道题需要先算 3+5=8，再算 8*2=16。
所以答案是 16。

草稿纸: 已算出 3+5=8，8*2=16，最终答案 16
```

下一轮用户问"你刚才怎么算的？"，草稿纸里的中间步骤就能用上。Scratchpad 特别适合多步推理、规划类任务——智能体先拆步骤，把每一步的结论写在草稿纸上，后面步骤就能引用前面的结果，不会乱。很多 ReAct 风格的智能体其实就是在做这件事，只不过它们把"思考"直接塞进 messages，咱们这里单独拆出来，更清晰。

## 六、整合：一个完整的记忆系统架构

把四种记忆拼在一起，数据流是这样的：

用户消息进来 → 从长期记忆回忆相关信息 → 加上工作记忆（草稿纸）→ 加上短期摘要 → 生成回答 → 更新短期记忆、工作记忆，必要时写入长期记忆。

下面是一个 `MemoryManager`，把四层记忆串起来：

```python
EXTRACT_MEMORY_PROMPT = """从以下用户消息中，提取值得长期记住的信息（如姓名、偏好、重要决定）。
若没有可记住的，输出"无"。
只输出提取的内容，不要解释。

用户消息：{content}
"""

class MemoryManager:
    def __init__(self, user_id):
        self.summary_mem = SummaryMemory(max_recent=6, summarize_threshold=12)
        self.long_term = LongTermMemory(user_id)
        self.working = WorkingMemory()

    def build_prompt(self, user_query, base_system_prompt):
        # 1. 从长期记忆回忆
        prompt = self.long_term.build_context(user_query, base_system_prompt)
        # 2. 加上工作记忆
        prompt = SCRATCHPAD_PROMPT.format(scratchpad=self.working.scratchpad) + "\n\n" + prompt
        return prompt

    def get_messages(self, user_query, base_system_prompt):
        full_system = self.build_prompt(user_query, base_system_prompt)
        return self.summary_mem.get_messages(full_system)

    def add_turn(self, user_msg, ai_response):
        self.summary_mem.add("user", user_msg)
        clean_response = self.working.update_from_response(ai_response)
        self.summary_mem.add("assistant", clean_response)
        # 提取关键信息存入长期记忆
        resp = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": EXTRACT_MEMORY_PROMPT.format(content=user_msg)}],
            temperature=0
        )
        extracted = resp.choices[0].message.content.strip()
        if extracted and extracted != "无":
            self.long_term.save_memory(extracted)
        return clean_response

# 完整对话流程示例
def chat_with_memory(user_msg, manager, base_prompt="你是一个贴心助手。"):
    msgs = manager.get_messages(user_msg, base_prompt)
    msgs.append({"role": "user", "content": user_msg})
    resp = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=msgs,
        temperature=0.7
    )
    ai_text = resp.choices[0].message.content
    return manager.add_turn(user_msg, ai_text)

# 模拟多轮对话
manager = MemoryManager(user_id="test_user")
chat_with_memory("我叫阿杰，是 Python 开发，最近在学 Go", manager)
chat_with_memory("Go 的并发模型和 Python 有啥区别？", manager)
# 模拟"跨会话"：新开一个 manager 但用同一个 user_id，长期记忆会保留
manager2 = MemoryManager(user_id="test_user")
ctx = manager2.long_term.build_context("推荐点学习资料", "你是助手。")
print("跨会话回忆:", ctx[:200])
```

**运行效果：**

```
跨会话回忆: 你是助手。

【关于这位用户的记忆】
- 最近在学 Go
- 职业 Python 开发
- 用户名阿杰
```

新会话里，AI 已经知道用户是谁、在学什么，推荐资料就能更精准。`add_turn` 里用 LLM 从用户消息里提取"值得长期记住"的信息，避免把闲聊也塞进向量库。如果你不想每次都调一次 API，可以改成规则：只当用户明确说"记住这个"时才写入长期记忆，或者用关键词触发。生产环境建议加个去重和过期机制，同一条记忆别重复存，太老的也可以定期清理。

## 七、对比总结

| 记忆类型 | 原理 | 持久性 | 适合场景 | 成本 |
|----------|------|--------|----------|------|
| 滑动窗口 | 保留最近 N 轮 | 会话内 | 简单对话 | 最低 |
| 摘要压缩 | LLM 总结历史 | 会话内 | 长对话、多步任务 | 中等 |
| 长期记忆 | 向量库持久化 | 跨会话 | 个性化助手 | 稍高 |
| 工作记忆 | Scratchpad 草稿 | 单次任务 | 复杂推理、多步计划 | 最低 |
